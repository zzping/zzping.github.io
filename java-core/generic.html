# Java 泛型

# 泛型简介

泛型（Generics），是 JDK5 提供的一个非常重要的新特性。

泛型从字面的意思理解就是泛化的类型，即参数化类型，相当于是把类型当作是参数一样传递！

Java 泛型的设计原则： 只要在编译期没有出现警告，那么在运行期就不会出现 ClassCastException 异常。

（1）为什么要使用泛型？

早期（JDK5 以前）Java 是用 Object 来代表任意类型的，但是在获取真实类型的时候需要强转，会有类型转换异常的风险，这样的程序是不安全和不稳定的。以集合为例，分析泛型给集合带来的好处：

集合没有使用泛型时：

// JDK5 之前的集合是没有泛型的, Collection、Map等集合对存储的元素类型没有任何限制, 对于集合来说他
// 都是使用 Object 来接收存入的元素, 因此在往集合中放入元素时, 什么类型都可以。也不会报错！
// 但是在使用 get() 获取集合中的元素的时候, 返回的是 Object, 需要强转才能原对象。

public void test01() {
   List list = new ArrayList();
   list.add("abc");
   list.add("xyz");
   list.add(100);
   list.add(true);
   System.out.println(list);

   for (int i = 0; i < list.size(); i++) {
       String s = (String) list.get(i); // 需要强转，可能会有类型转换异常
       System.out.println(s);
   }
}

public void test02() {
   List<String> list = new ArrayList<>();

   list.add("xyz");

       String s = list.get(i); // 不需要强转, 直接获取对象
       System.out.println(s);
   }
}

使用泛型的好处：

Java语言引入泛型的好处是安全简单。泛型的好处是在编译的时候检查类型安全，很重要的一点就是消除了强制类型转换，减少了出错的机会，让错误在编译阶段被提前发现，而不是在运行时报错！

（1）代码更加简介【不用强制转换】

（2）程序更加健壮【只要编译时期没有警告，那么运行时期就不会出现ClassCastException异常】

（3）可读性和稳定性【在编写集合的时候，就限定了类型】

（2）为什么泛型中的数据类型只能是引用数据类型，不能是基本数据类型？

因为虚拟机在编译时会把带泛型的转换成Object类型，而基本类型不属于Object类型，所以泛型里面数据类型不能是基本类型。

（3）什么是泛型擦除？

泛型只存在与编译期，泛型是提供给javac编译器使用的，它用于限定集合的输入类型，让编译器在源代码级别上，即挡住向集合中插入非法数据。但编译器编译完带有泛形的java程序后，生成的class文件中将不再带有泛形信息，以此使程序运行效率不受到影响，这个过程称之为“擦除”。

# 泛型的使用

# （1）泛型类

泛型类： 有一个或者多个类型变量的类。类上定义的类型变量可以在变量、方法中使用：

定义泛型类：

public class Parent<T> {
   private T data;

   public T getData() {
       return data;
   }

   public void setData(T data) {
       this.data = data;
   }
}

# （2）泛型接口

泛型接口： 有一个或者多个类型变量的接口，接口上定义的类型的变量可以用在方法中：

定义泛型接口：

public interface Generic1<T> {
public abstract void show(T t);
}

使用泛型接口：

（1）第一种情况： 实现类也是泛型类, 则实现类和接口的泛型类型要一致

class GenericImpl1<T> implements Generic1<T> {
   @Override
   public void show(T t) {}
}

class GenericImpl2<T, E> implements Generic1<T> {
   @Override
   public void show(T t) {}
}

# （3）泛型方法

泛型方法： 有一个或者多个类型变量的方法，方法中定义的类型的变量可以用在返回值、形参、方法体中：

public class Generic2 {

   // 定义泛型方法
   public <T> void test1(T t) {
       System.out.println(t);
   }

   // 定义泛型方法
   public <E> E test2(E e) {
       return e;
   }

   // 定义泛型方法
   public static <E> E test3(E e) {
       return e;
   }
}

泛型方法是独立于泛型类的，泛型方法可以存在与普通类中，也可以存在于泛型类中！

// 泛型方法的调用
public class TestGeneric {

   public static void main(String[] args) {
       // 当调用一个泛型方法时在方法名前的尖括号中放人具体的类型, 也可以省略, 会自动推断
       String a = Generic2.<String>test3("a");
       String b = Generic2.test3("b");
   }
}
