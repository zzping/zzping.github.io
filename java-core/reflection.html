# Java 反射

# 什么是反射

反射（Reflection）是 Java 中比较核心的技术。很多框架底层都用到了反射技术。那什么是反射呢？

反射指的是：对于任何一个类，在程序**运行的时候**都可以直接得到这个类的所有信息，并创建对象、调用对象的方法和属性。这种在运行时动态获取和修改程序数据的行为机制就称为反射。

  • 在运行时，可以获取到类的构造器对象（Constructor）
  • 在运行时，可以获取到类的成员变量对象（Field）
  • 在运行时，可以获取到类的成员方法对象（Method）
  • ......

使用反射首先需要得到 Class对象，通过 Class 对象我们可以创建对象实例，并调用对象的方法和属性。

# 获取 Class 对象的三种方式

（1）Class.forName("全限定类名") ： 多用于配置文件中，在配置文件中定义类名，读取配置文件加载类

（2）类名.class： 多用于参数的传递

（3）对象.getClass() 方法： 多用于在程序运行时通过对象实例获取对应的 Class 对象。

# 反射的四个核心 API

反射涉及到四个核心的类：

  • java.lang.Class： 类对象
  • java.lang.reflect.Constructor： 类的构造器对象
  • java.lang.reflect.Field： 类的成员变量对象
  • java.lang.reflect.Method： 类的成员方法对象

下面详细介绍使用反射过程的常用操作和核心 API 的使用：

Class 中常见的 API：

// 创建对象实例, 是通过调用对象的无参构造创建, JDK9 开始被废弃, 不推荐使用
// JDK9 开始推荐使用 clazz.getDeclaredConstructor().newInstance()
T newInstance();


// 获取简单类名
String getSimpleName();

// 获取全限定类名
String getName();

// 根据参数类型获取构造器(只能获取 public 修饰的构造器), 不用
Constructor<T> getConstructor(Class<?>... parameterTypes)
    
// 获取所有的构造器(只能获取 public 修饰的构造器)    
Constructor<?>[] getConstructors()
    
// 根据参数类型获取构造器(任意访问修饰符修饰的都可以), 推荐使用
Constructor<?>[] getDeclaredConstructors()

// 获取所有声明的构造器(任意访问修饰符修饰的都可以), 推荐使用
Constructor<T> getDeclaredConstructor(Class<?>... parameterTypes)
    
// 获取父类, Java 是单继承, 所以返回的不是数组类型    
public native Class<? super T> getSuperclass(); 

// 获取实现的接口, java 可以多实现, 所以返回的数组类型
public Class<?>[] getInterfaces() 
    

// 获取所有的注解
public Annotation[] getDeclaredAnnotations();

// 获取指定类型的注解
public <A extends Annotation> A[] getDeclaredAnnotationsByType(Class<A> annotationClass);
    
// 判断是否存在指定类型的注解      
public boolean isAnnotationPresent(Class<? extends Annotation> annotationClass);

# 反射的使用场景

Java 中的反射特性一般结合注解和配置文件（如 xml 配置文件）来使用。这也是大部分框架（Spring等）支持两种配置方式的原因。比如：

  • 如果是注解方式，服务端启动时，Spring框架会去扫描指定目录下的类，通过反射看类有没有Service注解，如果类上有 Service注解，会提前初始化（new）这个类。初始化好所有类以后，再去查找所有属性，看属性有没有Autowired注解，有的话，会给这个属性注入值（反射赋值）
  • 如果是 xml 的方式，原理上也差不多，只不过是先解析XML，拿到XML里的配置信息，再去初始化（new）或给属性反射赋值。所以我们写业务代码的时候才不用一个个的去new实现类，所有参数都赋上值，这部分工作Spring已经利用反射技术给完成了。

# 反射的优缺点

优点是：

  • 在程序运行期间可以操作对象的数据和方法，增加了程序的灵活性。
  • 反射结合注解和配置文件等方式能实现程序的解耦，可以更好的写出符合软件设计原则的代码。

缺点是：

  • 反射可以破坏面向对象的封装性。（可以通过 setAccessible(true) 方法来取消访问控制检查）
  • 反射可以破坏泛型的约束性（泛型工作在编译阶段，反射工作在运行阶段）。
@Test
public void test01() throws Exception {
   // 只能添加 String 类型的元素
   List<String> list = new ArrayList<>();
   list.add("a");
   list.add("b");

   System.out.println("=========================");

   // 1、先获取集合对象的 Class 对象
   Class c = list.getClass();
   // 2、从 ArrayList 集合中获取 add 方法
   Method add = c.getDeclaredMethod("add", Object.class);
   // 3、执行 list 集合的 add 方法, 这时候可以添加任意数据类型
   add.invoke(list, 1);
   System.out.println(list); // [a, b, 1]
}

# 总结

一句话概括反射： 在程序运行时可以获取任意一个类的属性、方法、类的所有信息，并可以动态的创建对象和调用对象的方法和属性。
