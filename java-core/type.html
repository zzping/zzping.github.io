# Java 类型（Type）

# Type 体系总览

Java 5 引入了泛型的概念，同时为了表示和处理泛型中的类型，增加 Type 体系。具体是增加了如下接口：

在 JDK5 之前，没有泛型，只有原始类型。此时所有的原始类型都通过字节码文件类 Class 类进行抽象。Class 类的一个具体对象就代表了一个指定的原始类型！

从 JDK5 开始，引入了泛型之后，同时也扩充了数据类型。从只有原始类型扩充到了如下 Type 体系：

  • java.lang.reflect.Type： 类型的顶层接口，是对 Java 中所有类型的一个抽象

  • java.lang.Class： 原始类型，也叫 raw type，是 Type 接口的实现类，反射的基础

  • java.lang.reflect.ParameterizedType： 参数化类型，带参数的类型，比如 List<String>

  • java.lang.reflect.TypeVariable： 类型变量，比如 List<E>中的 E，Map<K，V> 中的 K、V

  • java.lang.reflect.WildcardType： 泛型表达式类型，通配符类型（即带有 ? 的泛型参数），有三种：

  • List<?> 中的 ?

  • List<? extend Number> 中的 ? extend Number

  • List<? super Integer> 中的 ? super Integer

  • java.lang.reflect.GenericArrayType： 注意这里的数组类型并不是平常常见的 String[]、int[] 这种（这种都属于 Class 类型），这里的数组类型都是指带有泛型的数组，具体来讲指的是一种元素类型是ParameterizedType（参数化类型）或者TypeVariable（类型变量）的数组类型，如T[] 或者 List<E>[]

# Type（顶层接口）

java.lang.reflect.Type 接口是Java编程语言中所有类型的父接口，是对Java编程语言类型的一个抽象，源码如下：

public interface Type {

    default String getTypeName() {
        return toString();
    }
}

# ParameterizedType（参数化类型）

java.lang.reflect.ParameterizedType 接口代表参数化类型，即带参数的类型（即带 <> 的类型），比如： List<T>、

Map<K，V>、User<String> 等。ParameterizedType 接口的源码如下：

public interface ParameterizedType extends Type {
	
    // 获取参数类型<>里面的值,例如Map<K,V> 那么就得到 [K,V]的一个数组
    Type[] getActualTypeArguments();
	
    // 获取参数类型<>前面的值，例如例如Map<K,V> 那么就得到 Map
    Type getRawType();

    // 获取所有者类型（只有内部类才有所有者，比如Map.Entry他的所有者就是Map）, 不是内部类则返回null
    Type getOwnerType();
}

# TypeVariable（类型变量）

java.lang.reflect.TypeVariable 是一个接口，表示泛型的类型变量，比如 List<T> 中的 T、Map<K，V> 中的 K、V。实际的实现类是 TypeVariableImpl

注意： Map<String， T> 这种只有 T 属于 TypeVariable，而前面的 String 并不属于 TypeVariable

TypeVariable 接口的源码如下：

public interface TypeVariable<D extends GenericDeclaration> extends Type, AnnotatedElement {
    // 类型对应的上限, 默认为Object, 可以有多个
    Type[] getBounds();

    // 获取声明该类型变量实体，也就是TypeVariableTest<T>中的TypeVariableTest
    D getGenericDeclaration();

    // 获取类型变量在源码中定义的名称；
    String getName();
	
    // JDK8新增的
    AnnotatedType[] getAnnotatedBounds();
}

public class TypeVariableTest<T> {

    private T t;
    private List<T> list;
    private Map<String, T> map;

    public static void main(String[] args) {
        Class<TypeVariableTest> clazz = TypeVariableTest.class;

        for (Field field : fields) {
            System.out.println("属性名: " + field.getName());
            Type type = field.getGenericType();
            if (type instanceof TypeVariable) {
                TypeVariable typeVariable = (TypeVariable) type;
                System.out.println(typeVariable.getName());
                System.out.println(Arrays.asList(typeVariable.getBounds()));

                System.out.println("==========================================");
            }
            if (type instanceof ParameterizedType) {
                ParameterizedType parameterizedType = (ParameterizedType) type;
                Type[] typeArguments = parameterizedType.getActualTypeArguments();

                for (Type typeArgument : typeArguments) {
                    if (typeArgument instanceof TypeVariable) {
                        TypeVariable typeVariable = (TypeVariable) typeArgument;
                        System.out.println("name： " + typeVariable.getName());
                        System.out.println(typeVariable.getGenericDeclaration());
                        System.out.println("==========================================");
                    }
                }
            }
        }
    }
}

# WildcardType（通配符类型）

java.lang.reflect.WildcardType 是一个接口，表示通配符类型。指的是 List<?> 中的 ?、List< ? extend Number> 中的 ? extend Number、List< ? super Integer> 中的 ? super Integer 这种表达式。

注意： WildcardType 虽然是 Type 的一个子接口，但并不是 Java 类型中的一种，表示的仅仅是类似 ? extends Number、? super Integer 这样的通配符表达式。

WildcardType 接口源码如下：

public interface WildcardType extends Type {

    // 获取上边界
    Type[] getUpperBounds();

    // 获取下边界
    Type[] getLowerBounds();
}

WildcardType 演示代码：

public class WildcardTypeTest {

    private List<? extends Number> list1;
    private List<? super String> list2;

    public static void main(String[] args) {

        Class<WildcardTypeTest> clazz = WildcardTypeTest.class;
        Field[] fields = clazz.getDeclaredFields();

        for (Field field : fields) {
            System.out.println("属性名： " + field.getName());

            // 先获取到 List<? extends Number>
            ParameterizedType parameterizedType = (ParameterizedType) field.getGenericType();
            // 获取 List<? extends Number> 中的实际类型参数 ? extends Number
            Type[] typeArguments = parameterizedType.getActualTypeArguments();

            for (Type typeArgument : typeArguments) {
                System.out.println("实际的类型参数： " + typeArgument);
                if (typeArgument instanceof WildcardType) {
                    WildcardType wildcardType = (WildcardType) typeArgument;
                    System.out.println("上边界： " + Arrays.toString(wildcardType.getUpperBounds()));
                    System.out.println("下边界： " + Arrays.toString(wildcardType.getLowerBounds()));
                }
            }
            System.out.println("================================================");
        }
    }
}

# 7.6 GenericArrayType（泛型数组类型）

java.lang.reflect.GenericArrayType 是一个接口。表示泛型数组类型，具体指的是元素类型是ParameterizedType（参数化类型）或者TypeVariable（类型变量）的数组类型，如T[] 或者 List<E>[]

注意： 这里的数组不是我们平常说的 String[]、int[] 这种，而是指泛型数组。例如： T[]、List<E>[]

GenericArrayType 接口源码如下：

public interface GenericArrayType extends Type {
	
    // 返回泛型数组中元素的类型, 比如 T[] 中的 T、List<E>[] 中的 List<E>
    Type getGenericComponentType();
}

# 7.7 Class （原始类型）

java.lang.Class 是一个类，是 Type 接口唯一的直接子类，JDK1.0 就已经引入了 Class 类。它代表了Java 类型中的原始类型以及基本类型，同时它也是反射的基础，我们平常说的数组、枚举、接口、注解、字符串、整型等这些类型都是相对于 Class 来说的。

# 7.8 总结

（1）Java 中的所有类型

Java 中所有的类型包括：

（1）raw types：原始类型，对应 Class

（2）parameterized types：参数化类型，对应 ParameterizedType

（3）array types：数组类型，对应 GenericArrayType

（4）type variables：类型变量，对应 TypeVariable

（5）primitive types：基本类型，也是对应 Class

List、Map、Integer 指的就是 Class 类型

List<String> 是 ParameterizedType 类型、List<E> 中的 E 是 TypeVariable 类型、List<?> 中的 ? 是 WildcardType 类型、T[]、List<String>[] 就是 GenericArrayType 类型

（2）为什么要新增 Type 接口及一系列子接口 ？

JDK5 新增了 Type 接口以及 ParameterizedType, TypeVariable, GenericArrayType, WildcardType 四个属于 Type 的直接子接口。并将 Class 类作为 Type 接口的唯一的直接子类。目的就是为了处理泛型相关的各种类型。

为了程序的扩展性，最终引入了 Type 接口作为 Class 和 ParameterizedType，TypeVariable，GenericArrayType， WildcardType 这几种类型的总的父接口。这样可以用 Type 类型的参数来接受以上五种子类的实参或者返回值类型就是 Type 类型的参数。统一了与泛型有关的类型和原始类型 Class。

（3）为什么和泛型相关的类型不能统一到 Class 下 ？

Java 为了使用泛型又不真正引入泛型，Java采用泛型擦除机制来引入泛型。Java中的泛型仅仅是给编译器javac使用的，确保数据的安全性和免去强制类型转换的麻烦。但是，一旦编译完成，所有的和泛型有关的类型全部擦除。

因为泛型擦除的原因，与泛型有关的参数化类型、类型变量类型、限定符类型 、泛型数组类型这些类型编译后全部被打回原形，在字节码文件中全部都是泛型被擦除后的原始类型，并不存在和自身类型对应的字节码文件。所以和泛型相关的新扩充进来的类型不能被统一到Class类中。

（4）Type 接口中为什么没有需要重写的方法？

Type的出现仅仅起到了通过多态来达到程序扩展性提高的作用，没有其他的作用。

（5）反射获取类的泛型类型（实际开发实用）

实际开发中通过反射获取泛型类型，可以在泛型基类中这么写：

public class BaseDaoImpl<T> implements BaseDao<T> {

	// 它代表着实际类型
private Class<T> beanClass;

@SuppressWarnings("unchecked")
public BaseDaoImpl() {
  ParameterizedType parameterizedType=(ParameterizedType)this.getClass().getGenericSuperclass();
  beanClass=(Class<T>) parameterizedType.getActualTypeArguments()[0];
}
// 省略具体的操作....
}
