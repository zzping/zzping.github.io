# Java 注解

# 注解简介

概念：注解（Annotation）是 Java5 引入的新特性，注解可以标注在类、方法、变量、参数等位置上。

语法： @注解名称，例如 @Override

作用：注解的作用是标记，用于给 Java 代码提供元数据。

注解和反射一样，都是属于框架的底层基础技术。

# 注解分类

注解可以大致分为三类：JDK 内置的注解、第三方框架的注解、自定义注解

# （1）JDK 内置的注解

JDK 内置了一些注解，这些注解通常是为了用于给编译器一些指示，编译器根据不同的注解有相应处理。

（1）java.lang 包下有 5 个

  • @Override： 标注重写的方法，如果标注的不是重写的方法，编译器会报错
  • @Deprecated： 标记过时，可以标记类、方法、属性等等。使用过时的编译器会给出警告
  • @SuppressWarnings： 压制警告，指示编译器忽略指定的警告
  • @SafeVarargs： java7 新增，忽略任何使用参数为泛型变量的方法或构造函数调用产生的警告
  • @FunctionalInterface： java8 新增，只能标注在类上，用于标识该类是函数式接口

（2）java.lang.annotation 包下有 6 个

  • @Retention：表示注解的保留策略，有三种选择： .java 原文件中、.class 文件中、运行时
  • @Target： 标识注解作用在哪些目标上， 有类、方法、属性、参数、构造器等等
  • @Inherited：标记这个注解是继承于哪个注解类
  • @Documented：标记注解是否包含在用户文档中
  • @Repeatable： java8 新增，标识某注解可以在同一个声明上使用多次。
  • @Native： java8 新增，不常用

# （2）第三方框架的注解

很多第三方框架都会有自己的注解，比如：

（1）junit 框架： @Test、@Before、@After

（2）Spring 框架： @Controller、@Service、@Component

（3）Mybatis 框架： @Mapper、@Select、@Update

# （3）自定义注解

定义注解和定义类、接口、枚举方式一样，只不过定义注解用的是 @interface。比如自定义一个名为 MyTestAnnotation 的注解：

# （4）value 属性的特殊性

1、如果注解中只有一个 value 属性，那么使用注解时可以省略 value 名不写

2、如果注解中有多个属性并且包含 value 属性，并且除了 value 属性之外其他属性都有默认值，使用时只指定 value 属性时，则可以省略 value 名。

# 元注解（注解的保留策略、作用位置）

# （1）什么是元注解

元注解： 用来标记在注解上的注解

在自定义注解的时候可能会用到几个元注解：@Retention、@Target、@Documented

  • @Retention：表示注解的保留策略，有三种选择： .java 原文件中、.class 文件中、运行时
  • @Target： 标识注解作用在哪些目标上， 有类、方法、属性、参数、构造器等等
  • @Documented：标记注解是否包含在用户文档中， 不常用

# （2）注解的保留策略（@Retention）

注解的保留策略指的是，指的是注解会存在于什么阶段，.java 的源文件中、.class 文件中、还是在运行时还存在。这三种对应 RetentionPolicy 枚举的三个值： SOURCE、CLASS、RUNTIME。

注意： 自定义注解的保留策略基本都是 RUNTIME，因为只有 RUNTIME 才能在运行时通过反射获取。

@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {

}

# （3）注解的作用位置（@Target）

注解是一种标记，它可以标记在类上、方法上、属性上。通过 @Target 这个元注解来标识注解可以作用在什么位置。

@Target({ElementType.TYPE, ElementType.METHOD, ElementType.FIELD})
public @interface MyAnnotation {

}

   // 成员方法上

   // package 包上
   PACKAGE,

   // java8 新增的, 类型参数上
   TYPE_PARAMETER,

   // java8 新增的
   TYPE_USE
}

# 注解的本质

注解的是本质是一个接口（interface），通过 XJad 反编译工具可以看出：

1、先自定义一个注解

2、通过 javac MyAnnotation.java 命令编译得到 My Annotation.class 文件

3、通过反编译工具 XJad 打开 MyAnnotation.class 文件查看

Annotation 是所有注解的顶层接口，Java 中所有的注解都是继承自 Annotation 接口！就像所有的类都是继承 Object、所有的枚举都是继承 Enum、所有的异常都是 Throwable 的子类。

（3）使用反射获取注解信息

public class TestAnnotation {

   @Test
   public void test01() throws Exception {
       // 获取 Class 对象
       Class c = Student.class;
       if (c.isAnnotationPresent(MyAnnotation.class)) {
           MyAnnotation anno = (MyAnnotation) c.getAnnotation(MyAnnotation.class);
           System.out.println(anno);
           System.out.println("value: " + anno.value());
           System.out.println("description: " + anno.description());
       }

       // 反射获取 name 属性
       Field name = c.getDeclaredField("name");
       if (name.isAnnotationPresent(MyAnnotation.class)) {
           MyAnnotation anno = (MyAnnotation) c.getAnnotation(MyAnnotation.class);
           System.out.println(anno);
           System.out.println("value: " + anno.value());
           System.out.println("description: " + anno.description());
       }

       // 反射获取 hello 方法
       Method hello = c.getDeclaredMethod("hello");
       if (name.isAnnotationPresent(MyAnnotation.class)) {
           MyAnnotation anno = (MyAnnotation) c.getAnnotation(MyAnnotation.class);
           System.out.println(anno);
           System.out.println("value: " + anno.value());
           System.out.println("description: " + anno.description());
       }
   }
}

# 注解加反射简单模拟 junit 单元测试

效果： 自定义注解 @MyTest，只要标记了 @MyTest 的方法都可以自动触发执行。类似 Junit

（1）自定义注解 @MyTest

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface MyTest {

}
1
2
3
4
5

（2）解析注解并测试

public class TestAnnotation {

   @MyTest
   public void test01() {
       System.out.println("test01...");
   }

   @MyTest
   public void test02() {
       System.out.println("test02...");
   }

   @MyTest
   public void test03() {
       System.out.println("test03...");
   }

   public static void main(String[] args) throws Exception {
       // 创建一个对象, 用于触发方法执行的载体
       TestAnnotation testAnnotation = new TestAnnotation();
       // 获取 Class 对象
       Class c = TestAnnotation.class;
       // 获取对象所有的 public 修饰的方法
       Method[] methods = c.getMethods();
       for (Method method : methods) {
           // 如果方法上有 MyTest 注解, 就直接触发方法执行
           if (method.isAnnotationPresent(MyTest.class)) {
               method.invoke(testAnnotation);
           }
       }
   }
}
