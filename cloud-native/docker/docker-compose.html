# Docker Compose

# compose 简介

compose 是 Docker 官方的开源项目，它是用于定义和运行多容器 Docker 应用程序的工具。可以对多个容器进行编排（让多个容器有一定顺序的启动）。

通过 compose ，你可以使用 YAML 文件来配置应用程序的服务，然后通过一个命令即可从配置中创建并启动所有服务。

Github 地址： https://github.com/docker/compose

（1）compose 中两种重要的概念

  • 项目（project）： 由一组关联的应用容器组成的一个完整业务单元，在 docker-compose.yml 中定义
  • 服务（service）： 一个应用的容器，实际上可以包括若干运行相同镜像的容器实例。

# compose 安装和卸载

官方文档： compose 的安装 (opens new window)

安装前的准备工作：

  • docker compose 依靠 docker engine 才能工作。确保先安装了 docker engine
  • 桌面系统（mac、window）上，docker compose 是包含在了 docker engine 之中的，无序额外安装
  • linux 系统上，安装了 docker engine 后需要额外安装 docker compose

Linux 上安装 docker compose：

（1）通过 curl 下载 docker compose

curl -L "https://github.com/docker/compose/releases/download/1.28.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 放在 /usr/local/bin 这个目录就不用手动配置环境变量。随处都可以使用 docker-compose 命令

2
3

（2）修改 docker compose 文件的权限

chmod +x /usr/local/bin/docker-compose
1

（3）查看 docker compose 是否安装成功

docker-compose --version

# docker-compose version 1.28.2, build 67630359

Linux 上卸载 docker compose：

（1）如果是使用 curl 命令安装的，则卸载方法如下：

rm /usr/local/bin/docker-compose
1

（2）如果是使用 pip 命令安装的，则卸载方法如下：

pip uninstall docker-compose

# compose 的第一个程序

官方文档： 快速开始 (opens new window)

（1）新建一个空的目录 hello，并进入该目录下

mkdir hello
cd /hello
1
2

（2）新建一个 docker-compose.yml 文件

vim docker-compose.yml

# 添加如下内容
version: "3.8" # compose 的版本
services:
  tomcat: # 服务名,要保证唯一性
    image: tomcat:8.0.52 # 镜像
    container_name: tomcat01 # 容器名称
    ports: # 端口, 可以映射多个端口
      - 8080:8080

# compose 的 yml 文件模板指令

官方文档： compose 模板指令 (opens new window)

docker compose 的模板文件默认名称为 docker-compose.yml，格式为 YAML 格式。

注意： services 下的每一个服务都必须通过 image 指令指定镜像或者 build 指令（需要有 Dockerfile）来自动构建生成镜像。

# build

container_name 对应 docker run 命令中的 --name 参数

ports 对应 docker run 命令中的 -p 参数
