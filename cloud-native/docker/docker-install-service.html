# Docker 安装常用的软件

# Tomcat

官方文档： 使用 tomcat 镜像 (opens new window)

docker pull tomcat:8.0.52

docker run -p 8080:8080 --name tomcat01 \
-v tomcat01_apps:/usr/local/tomcat/webapps \
-v tomcat01_conf:/usr/local/tomcat/conf \
-d tomcat:8.0.52

错误使用方法：

docker run -p 8081:8080 --name tomcat02 
-v /data/tomcat/webapps:/usr/local/tomcat/webapps 
-v /data/tomcat/conf:/usr/local/tomcat/conf 
-d tomcat:8.0.52

注意： 上面的命令是错误的，容器启动之后马上又会退出。因为这种数据持久化的方式是 Bind mounts, 会使得 
/usr/local/tomcat/conf 文件夹中的文件都被隐藏。使用 docker logs tomcat2 查看日志：

    Feb 01, 2021 10:23:05 AM org.apache.catalina.startup.Catalina load
    WARNING: Unable to load server configuration from [/usr/local/tomcat/conf/server.xml]
    Feb 01, 2021 10:23:05 AM org.apache.catalina.startup.Catalina start
    SEVERE: Cannot start server. Server instance is not configured.
    
即找不到 /usr/local/tomcat/conf/server.xml 配置文件了。所以使用这种 Bind mounts 这种方式不能挂在 conf 目录。


只挂载 webapps 目录，同样，容器内 webapps 中的内容也会被隐藏

docker run -p 8081:8080 --name tomcat02 
-v /data/tomcat/webapps:/usr/local/tomcat/webapps 
-d tomcat:8.0.52

# MySQL

官方文档： 使用 mysql 镜像 (opens new window)

docker pull mysql:5.7.28

-v mysql01_data:/var/lib/mysql \
-v mysql01_conf:/etc/mysql \
-v mysql01_log:/var/log/mysql \

# Redis

官方文档： 使用 redis 镜像 (opens new window)

docker pull redis:5.0.10


// 启动一个 redis 容器，并开启 aof 持久化
// redis 容器的数据目录是 /data, 开启 aof 后默认数据都会在 /data/appendonly.aof
docker run -p 6379:6379 --name redis01 -d redis:5.0.10 redis-server --appendonly yes


// 使用自定义的配置文件启动 redis
mkdir -p /mydata/redis/conf
touch /mydata/redis/conf/redis.conf
在 redis.conf 中添加 appendonly yes 开启 aof 持久化

docker run -p 6379:6379 --name redis01 \ 
-v /mydata/redis:/usr/local/etc/redis \
-d redis:5.0.10 redis-server /usr/local/etc/redis/conf/redis.conf


注意： redis 的镜像中是没有 redis.conf 这个配置文件的。

# ElasticSearch

官方文档： 使用 elasticsearch 镜像 (opens new window)

// 拉取 elasticsearch 6.8.0 版本的镜像
docker pull elasticsearch:6.8.0

// 创建一个新的网桥, elk 都使用这个新的网络连接
docker network create elk_net

// 启动 elasticsearch
docker run --name es01 --net elk_net -p 9200:9200 -p 9300:9300  \
-v es_data:/usr/share/elasticsearch/data \
-v es_config:/usr/share/elasticsearch/config \
-v es_plugins:/usr/share/elasticsearch/plugins \
-d elasticsearch:6.8.0


// 第一次启动 elasticsearch 会遇到下面的问题：提示最大虚拟内存太小，至少需要 262144
bootstrap checks failed max virtual memory areas vm.max_map_count [65530] likely too low, increase to at least [262144]

// 修改宿主机系统配置, 调整虚拟内存大小
vim /etc/sysctl.conf

// 加入如下配置
vm.max_map_count=262144

// 启用配置
sysctl -p

// 重新启动容器
docker start es01

// 启动成功后, 通过浏览器访问 http://192.168.145.31:9200 看到 es 返回的 json 数据说明安装成功

# Kibana

官方文档： 使用 kibana 镜像 (opens new window)

// 拉取 kibana 6.8.0 版本的镜像
docker pull kibana:6.8.0

// 启动方式1： 通过指定环境变量方式启动
docker run -p 5601:5601 --net elk_net --name kibana01 \
-e ELASTICSEARCH_URL=http://192.168.145.31:9200 \
-d kibana:6.8.0

// 启动方式2： 通过加载配置文件方式启动
docker run -p 5601:5601 --net elk_net --name kibana01 \
-v kibana_config:/usr/share/kibana/config \
-d kibana:6.8.0

// 启动后查看并修改 kibana 的配置文件 
vim /var/lib/docker/volumes/kibana_config/_data/kibana.yml

// 修改配置文件中的 es 的地址【注意要把 [] 两边的空格去掉】
elasticsearch.hosts: ["http://192.168.145.31:9200"]

// 重新启动 kibana 容器
docker restart kibana01

// 打开浏览器, 查看 http://192.168.145.31:5601
