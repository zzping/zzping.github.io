# Docker 常用命令

Docker 官方命令大全： https://docs.docker.com/reference/

# 系统帮助命令

（1）docker version： 查询 docker 版本

（2）docker info： 查看 docker 详细信息

（3）docker --help：帮助命令

（4）docker image --help： 查看单个命令的帮助说明

（5）docker： 和 docker --help 一样用于显示 docker 的命令帮助

（6）docker system： docker 管理相关命令（通过 docker system -help 得知它包含四个子命令）

docker system df 查看 docker 整体磁盘占用情况

docker system info

docker system prune

docker system events

# 镜像相关命令

（1）查看镜像： docker image ls / docker images

docker images -a 列出所有镜像，包含中间层镜像 docker images -q 列出所有镜像的 id

docker images tomcat 列出 tomcat 的所有版本的镜像

（2）拉取镜像： docker image pull / docker pull

docker pull tomcat 拉取最新版本的 tomcat，等价于 docker pull tomcat:latest

docker pull tomcat:8.5.61 拉取 8.5.61 版本的 tomcat 镜像

（3）删除镜像： docker image rm / docker rmi

docker rmi tomcat:8.5.61 删除 8.5.61 版本的镜像

docker rmi -f tomcat 强制删除 tomcat 镜像

docker rmi -f $(docker images -qa) 强制删除所有的镜像

（4）查看镜像细节： docker image inspect / docker inspect

docker inspect 镜像id 查看镜像的内部详细信息

（5）导入镜像： docker image load / docker load

【选项】

-i	指定导入 tar 格式的文件
1

【用法】

docker load -i xxx.tar 将 xxx.tar 导入成镜像文件并放到本地镜像仓库中

（6）导出镜像： docker image save / docker save

【作用】 将 docker 镜像导出为 tar 文件

【用法】 docker save 镜像名 -o 名称.tar

# 容器相关命令

（1）查看容器： docker container ls / docker ps

【选项】

-a		查看所有的容器（正在运行的 + 没有运行的），-a 是 --all 的缩写
-q		查看所有正在运行的容器的id，-q 是 --quiet 的缩写
1
2

【示例】

docker ps				 查看正在运行的容器
docker ps -a			查看所有的容器
docker ps -q			查看所有正在运行的容器的 ID
docker ps -qa		 查看所有容器（正在运行 + 未运行）的 ID
1
2
3
4

（2）运行容器：docker container run / docker run

官方的运行容器的命令选项大全： docker run (opens new window)，也可以通过 docker run --help 获取查看该命令的选项参数

【选项】

-p				端口映射，-p 宿主机端口:容器内端口，-p 是 --publish 的缩写。可以写多个 -p，代表映射多个端口
-e				设置环境变量，-e 是 --env 的缩写
-d				容器以后台方式运行并返回容器 ID，默认是以前台方式启动
-v				数据卷，实现宿主机的文件/目录和容器内的文件/目录的共享
--name	 指定容器的名字

-v 宿主机目录:容器目录
1
2
3
4
5
6
7

【示例】

docker run -p 8080:8080 --name tomcat01 -d tomcat:8.0.52

（6）删除容器： docker container rm / docker rm

【选项】

-f		强制删除容器（可以删除正在运行的容器），-f 是 --force 的缩写。
1

【示例】

docker rm 容器id		删除容器，如果该容器正在运行，则会报错
docker rm -f 容器id		强制删除容器，可以删除正在运行的容器
docker rm -f $(docker ps -qa)		强制删除所有的容器

（11）容器和宿主机之间复制文件： docker container cp / docker cp

【用法】

（1）从容器中复制文件到宿主机：

docker cp 容器ID:容器内的路径  宿主机路径
1

（2）从宿主机复制文件到容器内：

docker cp 文件|目录  容器ID:容器路径
1

【示例】

（1）将容器内的目录/文件复制到宿主机

# 将 tomcat01 容器中的 webapps 目录下的 tt.txt 文件复制到宿主机的当前目录下
docker cp tomcat01:/usr/local/tomcat/webapps/tt.txt ./

# 将 tomcat01 容器中的 webapps 目录复制到宿主机的当前目录下
docker cp tomcat01:/usr/local/tomcat/webapps/ ./
1

（2）将宿主机的目录/文件复制到容器内

# 将宿主机当前目录下的 a.txt 文件复制到 tomcat01 容器下的 tomcat 目录下
docker cp ./a.txt tomcat01:usr/local/tomcat/

# 将宿主机当前目录下的 a.txt 文件复制到 tomcat01 容器下的 tomcat 目录下并重命名为 b.txt
docker cp ./a.txt tomcat01:usr/local/tomcat/b.txt

# 将宿主机的当前目录中的 test 目录复制到 tomcat01 容器的 tomcat 目录下
docker cp test/ tomcat01:/usr/local/tomcat/

（13）容器打包成镜像： docker container commit / docker commit

【选项】

-a 作者信息

-m 提交信息

-p 提交的时候暂停容器，默认的值就是 ture。即将当前容器提交成镜像的时候，容器会有短暂的暂停状态

【示例】

docker commit -a "zzp" -m "我的第一个 tomcat 镜像" 容器id mytomcat:1.0.0
