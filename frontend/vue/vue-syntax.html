# ES6 语法补充

# ES6 简介

ECMAScript 6.0（以下简称 ES6）是 JavaScript 语言的下一代标准，已经在 2015 年 6 月正式发布了。它的目标，是使得 JavaScript 语言可以用来编写复杂的大型应用程序，成为企业级开发语言。

# ECMAScript 和 JavaScript 的关系

一个常见的问题是，ECMAScript 和 JavaScript 到底是什么关系？

要讲清楚这个问题，需要回顾历史。1996 年 11 月，JavaScript 的创造者 Netscape 公司，决定将 JavaScript 提交给标准化组织 ECMA，希望这种语言能够成为国际标准。次年，ECMA 发布 262 号标准文件（ECMA-262）的第一版，规定了浏览器脚本语言的标准，并将这种语言称为 ECMAScript，这个版本就是 1.0 版。

该标准从一开始就是针对 JavaScript 语言制定的，但是之所以不叫 JavaScript，有两个原因。一是商标，Java 是 Sun 公司的商标，根据授权协议，只有 Netscape 公司可以合法地使用 JavaScript 这个名字，且 JavaScript 本身也已经被 Netscape 公司注册为商标。二是想体现这门语言的制定者是 ECMA，不是 Netscape，这样有利于保证这门语言的开放性和中立性。

因此，ECMAScript 和 JavaScript 的关系是，前者是后者的规格，后者是前者的一种实现（另外的 ECMAScript 方言还有 JScript 和 ActionScript）。日常场合，这两个词是可以互换的。

# let 关键字

ES6 新增了 let 关键字用来声明变量，它的用法类似于 var，但是使用 let 声明的变量，只在 let 关键字所在的代码块内有效。

let 关键字的解读： 阮一峰的 ECMAScript 6 入门 (opens new window)

{
    var a = 10;
    let b = 20;
}
console.log(a); // 10
console.log(b); // Uncaught ReferenceError: b is not defined

（1）作用域的概念

ES5 只有全局作用域和函数作用域，没有块级作用域（比如在 if 和 for 代码块中）。会有很多不合理的场景。

ES6 在全局作用域和函数作用域的基础上 ，增加一个块级作用域（let 关键字让声明的变量有了块级作用域）。

总结：

ES5 中的 var 是没有块级作用域的（if / for）

ES6 中的 let 是由块级作用域的（if / for）

ES6 之前因为 if 和 for 都没有块级作用域的概念，所以在很多时候，我们都必须借助 fucntion 的作用域来解决引用外面变量的问题。ES6 中加入了 let 关键字，它让变量有了块级作用域（主要是 if 和 for 块级），就可以使用 ES6 中的 let 来解决问题。

# const 关键字

ES6 新增了 const 关键字用来一个只读的常量。一旦声明，常量的值就不能改变。

（1）一旦声明，常量的值就不能修改

const a = 10;
console.log(a);
a = 20 // Uncaught TypeError: Assignment to constant variable.

（2）必须在声明的时候进行赋值

const a;

obj.name = "kobe" // 对象内部的属性值可以改变
obj.age = 42

obj = {} // Uncaught TypeError: Assignment to constant variable. 对象的值(引用地址)不能改变

# v-cloak

v-cloak： cloak 是斗篷、披风的意思，

有时候因为加载延时问题，例如卡掉了，数据没有及时刷新，就造成了页面显示从到message变量“你好啊”的变化，这样闪动的变化，会造成用户体验不好。此时可以使用v-cloak配置 display: none解决这个问题。

原理是： html 从上往下加载，vue 完成解析之前，v-cloak 会被当成标签的一个属性，这个属性有 display: none 的样式，所以被隐藏。vue 完成解析之后，v-cloak 属性就不会存在了。就可以正常显示解析后的数据。

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Title</title>
  <style>  
    [v-cloak] {
      display: none;
    }
  </style>
</head>
<body>
<div id="app">
  <h2>{{msg}}</h2>
  <h2 v-cloak>{{msg}}</h2>
</div>

<script src="../js/vue.js"></script>
<script>
  const vm = new Vue({
    el: '#app',
    data: {
      msg: "Hello Vue.js"
    }
  })
</script>
</body>
</html>

<!-- 浏览器F12 点击 Network 可以选择网络是 3G 模式进行模拟插件闪烁问题 -->

# 动态绑定

前面的插值语法主要作用是将数据插入到我们 模板的内容 当中。但是除了内容需要动态的决定以外，某些属性我们也希望可以动态绑定。比如

  • img 标签的 src 属性
  • a 标签的 href 属性
  • 标签的 style 属性
  • 标签的 class 属性

这个时候我们就可以使用 v-bind 指定。

# v-bind 的基本用法

<div id="app">

  <!-- <img v-bind:src="{{imgURL}}" alt=""> -->
    
  <!-- 正确的做法使用v-bind指令 -->
  <img v-bind:src="imgURL" alt="">
  <a v-bind:href="aHerf">百度一下</a>
    
  <!-- 语法糖写法(v-bind 的简写) -->
  <img :src="imgURL" alt="">
  <a :href="aHerf">百度一下</a>
</div>

# 计算属性

模板内的表达式非常便利，但是设计它们的初衷是用于简单运算的。在模板中放入太多的逻辑会让模板过重且难以维护。例如：

<div id="example">
  {{ message.split('').reverse().join('') }}
</div>
1
2
3

在这个地方，模板不再是简单的声明式逻辑。你必须看一段时间才能意识到，这里是想要显示变量 message 的翻转字符串。当你想要在模板中的多处包含此翻转字符串时，就会更加难以处理。

所以，对于任何复杂逻辑，你都应当使用 计算属性 (computed) 。

# 计算属性的基本使用

<div id="app">
  <!-- 使用 Mustache 语法进行拼接 -->
  <h2>{{firstName + " " + lastName}}</h2>
  <h2>{{firstName}} {{lastName}}</h2>

  <!-- 使用 methods 中的方法 -->
  <h2>{{getFullName()}}</h2>

  <!-- 使用计算属性 -->
  <h2>{{fullName}}</h2>
</div>

<script src="../js/vue.js"></script>
<script>
  const vm = new Vue({
    el: '#app',
    data: {
      firstName: "Lebron",
      lastName: "Jams"
    },
    methods: {
      getFullName: function () {
        return this.firstName + " " + this.lastName
      }
    },
    // 计算属性
    computed: {
      // 这里名字最好是名词，因为这里写的是计算属性
      fullName: function () {
        return this.firstName + " " + this.lastName
      }
    }
  })
</script>

<div id="app">

<script src="../js/vue.js"></script>
<script>
  const vm = new Vue({
    el: '#app',
    data: {
      books: [
        {id: 101, name: 'Unix编程艺术', price: 128},
        {id: 102, name: '代码大全', price: 110},

        {id: 104, name: '现代操作系统', price: 78}
      ]
    },
    computed: {
      totalPrice: function () {
        /* ES5 语法 (fori) */
        /*let sum = 0
        for (let i = 0; i < this.books.length; i++) {
          sum += this.books[i].price
        }
        return sum*/


        /* ES5 语法(for in) */
        /*let sum = 0
        for (let i in this.books) {
          sum += this.books[i].price
        }
        return sum*/


        /* ES6 语法 (for of)*/
        /*let sum = 0
        for (let book of this.books) {
          sum += book.price
        }
        return sum*/


        /* ES6 语法 (map、reduce) */
        return this.books.map(item => item.price).reduce((p1, p2) => p1 + p2)
      }

# 计算属性的 getter 和 setter

问题： 计算属性 computed 中写的是一个方法，为什么使用的时候是把它当成一个属性来使用？

# 计算属性的缓存

计算属性是有缓存的，这也就是为什么对于一些复杂的计算推荐使用计算属性而不是 methods。

<div id="app">
    <!-- 方式1： 直接拼接： 语法过于繁琐 -->
    <h2>{{firstName}} {{lastName}}</h2>

    <!-- 方式2： 通过定义 methods -->
    <h2>{{getFullName()}}</h2>
    <h2>{{getFullName()}}</h2>
    <h2>{{getFullName()}}</h2>

    <!-- 方式3： 通过定义计算属性 computed -->
    <!-- 多次调用, 只要 firstName 和 lastName 的值没有变化, 就只会执行一次，其他时候都走缓存 -->
    <h2>{{fullName}}</h2>
    <h2>{{fullName}}</h2>
    <h2>{{fullName}}</h2>
</div>

<script src="../js/vue.js"></script>
<script>
    const vm = new Vue({
        el: '#app',
        data: {
            firstName: "Kobe",
            lastName: "Bryant"
        },
        methods: {
            getFullName: function () {
                console.log("getFullName");
                return this.firstName + " " + this.lastName
            }
        },
        computed: {
            fullName: function () {
                console.log("fullName");
                return this.firstName + " " + this.lastName
            }
        }
    })
</script>

# v-on 的修饰符

（1）事件修饰符

在事件处理程序中调用 event.preventDefault() 阻止事件的默认行为 或 event.stopPropagation() 停止事件冒泡 是非常常见的需求。尽管我们可以在方法中轻松实现这点，但更好的方式是：方法只有纯粹的数据逻辑，而不是去处理 DOM 事件细节。

为了解决这个问题，Vue.js 为 v-on 提供了事件修饰符。之前提过，修饰符是由点开头的指令后缀来表示的。

<div id="app">

 <div style="width: 100px;height: 100px;background: blue" @click.stop="test2">		</div>
</div>
<!-- (2) 阻止事件默认行为 -->
<a href="http://www.baidu.com" @click.prevent="test3">点击一下</a>
</div>

<script type="text/javascript" src="../js/vue.js"></script>
<script type="text/javascript">
new Vue({
el: "#app",
methods: {
 test1() {
     alert("outer box")
 },
 test2(event) {
     // event.stopPropagation()
     alert("inner box")
 },
 test3(event) {
     // event.preventDefault()
     alert("test3 被点击了")
 }
}
})
</script>


<!-- 提交事件不再重载页面 -->

<!-- 修饰符可以串联 -->

# 检测数组更新

因为 Vue 是响应式的，Vue 内部会监听数据的变化，一旦数据发生变化，视图会发生对应的更新。Vue 将被监听的数组的变更方法进行了包裹，以保证当调用这些数组的相关方法后，视图可以自动更新。具体方法如下：

  • push()
  • pop()
  • shift()
  • unshift()
  • splice()
  • sort()
  • reverse()

有一些数组的方法 Vue 内部是没有进行包裹的，所以调用这些没有被包裹的方法不会自动更新视图，例如：

  • filter()
  • concat()
  • slice()
  • 直接使用数组的索引修改元素（this.names[0] = 'kobe'）

# 图书购物车

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Title</title>
  <style>
    table {
      border: 1px solid #e9e9e9;
      border-collapse: collapse;
      border-spacing: 0;
    }

    th, td {
      padding: 8px 16px;
      border: 1px solid #e9e9e9;
      text-align: left;
    }

    th {
      background-color: #f7f7f7;
      color: #5c6b77;
      font-weight: 600;
    }
  </style>
</head>
<body>
<div id="app">
  <div v-if="this.books.length > 0">
    <table border="10">
      <tr>
        <th></th>
        <th>书籍名称</th>
        <th>出版日期</th>
        <th>价格</th>
        <th>购买数量</th>
        <th>操作</th>
      </tr>
      <tr v-for="(item,index) in books">
        <td>{{item.id}}</td>
        <td>{{item.name}}</td>
        <td>{{item.publishDate}}</td>
        <td>{{item.price | formatPrice}}</td>
        <td>
          <button @click="decrement(index)">-</button>
          {{item.amount}}
          <button @click="increment(index)">+</button>
        </td>
        <td>
          <button @click="removeBook(index)">移除</button>
        </td>
      </tr>
    </table>
    <h2>总价格： {{totalPrice | formatPrice}}</h2>
  </div>
  <h2 v-else>购物车空空~</h2>
</div>

<script src="../js/vue.js"></script>
<script>
  const vm = new Vue({
    el: '#app',
    data: {
      books: [
        {id: 1, name: '《算法导论》', publishDate: '2006-09', price: 80, amount: 1},
        {id: 2, name: '《Unix编程艺术》', publishDate: '2006-02', price: 120, amount: 1},
        {id: 3, name: '《编程珠玑》', publishDate: '2008-05', price: 99, amount: 1},
        {id: 4, name: '《代码大全》', publishDate: '2006-03', price: 89, amount: 1}
      ]
    },
    methods: {
      // 增加数量
      increment(index) {
        this.books[index].amount += 1
      },
      // 减少数量
      decrement(index) {
        if (this.books[index].amount > 1) {
          this.books[index].amount -= 1
        }
      },
      // 移除图书
      removeBook(index) {
        this.books.splice(index, 1)
      }
    },
    computed: {
      // 计算购物车总价格
      totalPrice() {
        return this.books
            .map(item => item.price * item.amount)
            .reduce((p1, p2) => p1 + p2)
      }
    },
    // 过滤器
    filters: {
      formatPrice(price) {
        return '￥' + price.toFixed(2)
      }
    }
  })
</script>
</body>
</html>

# v-model 的原理

v-model 其实是一个语法糖，它的背后本质上是包含两个操作：

  • v-bind 指令绑定一个 value 属性
  • v-on 指令给当前元素绑定 input 事件
<div id="app">
    <!-- v-model 实现数据的双向绑定 -->
    <input type="text" v-model="msg1">
    <h2>{{msg1}}</h2>

    <!-- v-bind 结合 v-on 实现双向绑定 -->
    <input type="text" v-bind:value="msg2" v-on:input="msg2 = $event.target.value">
    <h2>{{msg2}}</h2>
</div>

<script src="../js/vue.js"></script>
<script>
    const vm = new Vue({
        el: '#app',
        data: {
            msg1: 'Hello',
            msg2: 'Vue.js'
        }
    })
</script>

# v-model 修饰符

（1）lazy 修饰符

在默认情况下，v-model 在每次 input 事件触发后将输入框的值与数据进行同步。可以添加 lazy 修饰符，从而转为在 change 事件之后进行同步。

<!-- 在“change”时(失去焦点、按 enter 键)而非“input”时更新 -->
<input v-model.lazy="msg">
1
2

（2）number 修饰符

如果想自动将用户的输入值转为数值类型，可以给 v-model 添加 number 修饰符。这通常很有用，因为即使在 type="number" 时，HTML 输入元素的值也总会返回字符串。如果这个值无法被 parseFloat() 解析，则会返回原始的值。

<!-- number 修饰符 -->
<input v-model.number="age" type="number">
1
2

（3）trim 修饰符

如果要自动过滤用户输入的首尾空白字符，可以给 v-model 添加 trim 修饰符

<!-- trim 修饰符 -->
<input v-model.trim="msg">
