# Vue Router（路由）

# 路由的概述

# 路由是什么

路由（routing）是网络工程中的一个常见术语。指的是通过互联的网络把信息从源地址传输到目的地址的活动。简单理解路由就是将信息或者数据从一个地方转移到另一个地方，指的是一种映射关系（url 和页面的映射）。

# 从后端路由到前端路由

精读前后端渲染之争 (opens new window)

浅谈前后端路由与前后端渲染 (opens new window)

（1）后端路由： url 和 页面的映射由后端管理（匹配 url 然后跳转到指定的页面或者路径）

（2）前端路由： url 和 页面（组件）的映射由前端管理

（1）后端渲染：后端将页面包括数据渲染后，直接返回给前端进行展示

（2）前端渲染：后端提供数据，前端拿到数据通过 JS 进行渲染展示

前端路由的核心： 改变 URL，但是页面不进行整体的刷新。

# （1）URL 的 hash

URL的hash也就是锚点(#), 本质上是改变window.location的href属性
我们可以通过直接赋值location.hash来改变href, 但是页面不发生刷新

hash 模式利用的是浏览器不会对#号后面的路径对服务端发起路由请求。即 HTTP 请求中不包括 # 
window.location.hash读取#值


优点:  兼容性高
缺点:  URL带有#号不好看

# （2）HTML5 的 history

HTML5 提供了一个 API:  history, 表示当前窗口的浏览历史。
浏览器工具栏的“前进”和“后退”按钮，其实就是对 History 对象进行操作。


优点:  URL不带#号, 好看
缺点:  是既需要浏览器支持也需要后端服务器支持


window.history属性指向 History 对象，它表示当前窗口的浏览历史。
	- history 的属性：
		- length
		- state
	- history 的方法：
		- back()、forward()、go()  ====> 这三个方法用于在浏览历史之中移动。
		- pushState() ===> 在历史中添加一条记录。
		- replaceState() ===>  修改当前记录

	(4) history.pushState({}, '', 'bbb')
	(5) history.back()
	(6) history.forward()

# （5）路由懒加载

当打包构建应用时，JavaScript 包会变得非常大，影响页面加载。如果我们能把不同路由对应的组件分割成不同的代码块，然后当路由被访问的时候才加载对应组件，这样就更加高效了。

路由的懒加载就是将路由对应的组件打包成一个个的 JS 代码块。这样只有在某个路由被访问时才加载对应的组件。这样更加高效。路由懒加载的实现方案是Vue 的异步组件 + Webpack 的代码分割功能。

# （9）导航守卫

导航守卫就是路由跳转过程中的一些钩子函数，路由跳转可以分为几个过程，这个大的过程分为跳转前中后等等细小的过程，在每一个过程中都有一函数，这个函数能让你操作一些其他的事儿的时机，这就是导航守卫。

（1）导航守卫分类：全局导航守卫、路由独享的导航守卫、组件内的导航守卫

（1）全局导航守卫： 指路由实例上直接操作的钩子函数，他的特点是所有路由配置的组件都会触发

钩子函数按照顺序包括 
	- beforeEach
	- beforeResolve(2.5.0+)
	- afterEach


用法如下：
const router = new VueRouter({ ... })
router.beforeEach((to, from, next) => {
// ...
})
