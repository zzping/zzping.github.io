# 认识组件化

# 什么是组件化

我们将一个完整的页面分成很多个组件。每个组件都用于实现页面的一个功能块。而每一个组件又可以进行细分。

各个组件还可以方便复用和维护管理。

# Vue 的组件化思想

组件化是 Vue.js 中的重要思想。它提供了一种抽象，让我们可以开发出一个个独立可复用的小组件来构造我们的应用。任何的应用都会被抽象成一颗组件树。

# 全局组件和局部组件

组件注册的时候有两种注册类型：

  • 全局注册： 使用Vue.component()注册
  • 局部注册： 使用 Vue 实例选项中的 components属性

（1）注册全局组件

// 定义一个组件
const myComponent = Vue.extend({
    template: `<div><h2>我是标题</h2><p>我是内容</p></div>`
})

// 注册全局组件
Vue.component('myComponent', myComponent)

# 注册组件的语法糖（简化写法）

前面使用组件的方式实际上是比较繁琐的，要先调用 Vue.extends()，然后通过 Vue.component() 注册组件。实际上 Vue2 开始很少见到这种写法了。Vue 为了简化组件使用步骤，省略了调用 Vue.extends() 的这一步，可以直接使用一个对象进行代替。下面演示注册全局组件和局部组件的常用简化写法：

<div id="app">
    <my-component1></my-component1>
    <my-component2></my-component2>
</div>

<script src="../js/vue.js"></script>
<script>
    // 注册全局组件
    Vue.component('MyComponent1', {
        template: `<div><h2>我是标题1</h2><p>我是内容1</p></div>`
    })
    const vm = new Vue({
        el: '#app',
        data: {},
        components: {
            // 注册局部组件
            MyComponent2: {
                template: `<div><h2>我是标题2</h2><p>我是内容2</p></div>`
            }
        }
    })
</script>

# 组件和模板的分离写法

上面的组件使用的写法中，组件的定义中还有模板代码（template 的值是 html 代码），代码看起来非常乱，我们可以将组件和模板进行分开来写。Vue 中有两种分离的写法

  • script 标签
  • template 标签
<div id="app">
    <my-component1></my-component1>
    <my-component2></my-component2>
</div>
<!-- 方式一： 使用 script 标签 -->
<script type="text/x-template" id="myComponent1">
  <div>
    <h2>我是标题</h2>
    <p>我是内容</p>
    </div>
</script>

<!-- 方式二： 使用 template 标签 -->
<template id="myComponent2" id="myComponent2">
    <div>
        <h2>我是标题</h2>
        <p>我是内容</p>
    </div>
</template>

<script src="../js/vue.js"></script>
<script>
    // 全局注册
    Vue.component('MyComponent1', {
        template: '#myComponent1'
    })
    const vm = new Vue({
        el: '#app',
        data: {},
        components: {
            // 局部注册
            MyComponent2: {
                template: '#myComponent2'
            }
        }
    })
</script>

# 父子组件的通信

在开发中，往往一些数据需要从上层传递到下层。比如在一个页面中，我们从服务器请求了很多数据，其中一部分数据并不是我们整个页面的大组件来显示的，而是需要下面的子组件进行展示。这个时候并不会让子组件再次发送网络请求，而是直接让大组件（父组件）将数据传递给小组件（子组件）。

如何让父子组件进行通信，分为两种情况：

  • 父组件向子组件传递数据： 通过 props
  • 子组件向父组件传递数据： 通过 $emit 事件

实际开发中，Vue实例和子组件通信、父组件和子组件通信过程是一样的。

# （1）父组件向子组件传递数据 - props

props 是 properties 的简写，属性的意思。即通过 v-bind 绑定自定义属性，然后通过 props 接收自定义的属性。

<div id="app">
    <!-- 在父组件中通过 v-bind 将数据传递给子组件-->
    <my-component :books="books" :msg="msg"></my-component>
</div>

<template id="myComponent">
    <div>
        <h2>我是子组件</h2>
        <p>{{msg}}</p>
        <ul>
            <li v-for="item in books">{{item}}</li>
        </ul>
    </div>
</template>
<script src="../js/vue.js"></script>
<script>

    // 定义子组件
    const MyComponent = {
        template: '#myComponent',
        // 在子组件中通过 props 接收数据
        props: ['books','msg']
    }

    // 将 Vue 实例看做父组件
    const vm = new Vue({
        el: '#app',
        data: {
            msg: 'Hello Vue.js',
            books: ['西游记', '红楼梦', '水浒传', '三国演义']
        },
        // 在父组件中注册子组件(下面的写法是 ES6 的写法)
        components: {
            MyComponent
        }
    })
</script>

prop 的大小写问题：

HTML 中的属性名是大小写不敏感的，浏览器会把所有的大写字符解释为小写字符。在 Vue 的写法中，推荐的是在写 props 的时候，使用驼峰命名（postList），而在 HTML 中使用全小写字母并且中划线分割的格式（post-list）

<my-component :post-list="postList"></my-component>

# （2）子组件向父组件传递数据 - $emit

有时候我们也需要在子组件中向父组件中传递数据，这时候我们使用自定义事件。

  • 在子组件中，通过 $emit() 来触发事件
  • 在父组件中，通过 v-on 来监听子组件事件（v-on不仅可以监听DOM事件，也可以监听组件间的自定义事件）
<!-- 2 父组件模板 -->
<div id="app">
    <!-- 这里方法不传递参数默认传递的不是 事件对象而是实际参数 item -->
    <my-component @categoryclick="categoryclick"></my-component>
</div>

<!-- 1 子组件模板 -->
<template id="myComponent">
    <div>
        <button v-for="item in categories" @click="categoryclick(item)">
            {{item.name}}
        </button>
    </div>
</template>
<script src="../js/vue.js"></script>
<script>

    // 1 子组件
    const MyComponent = {
        template: '#myComponent',
        data() {
            return {
                categories: [
                    {id: '101', name: '美妆护肤'},
                    {id: '102', name: '钟表珠宝'},
                    {id: '103', name: '电脑办公'},
                    {id: '104', name: '运动户外'}
                ]
            }
        },
        methods: {
            categoryclick(item) {
                // console.log(item);
                this.$emit('categoryclick', item)
            }
        }
    }

    // 2 父组件
    const vm = new Vue({
        el: '#app',
        data() {
            return {

            }
        },
        methods: {
            categoryclick(item) {
                console.log(item);
            }
        },
        components: {
            MyComponent
        }
    })
</script>

# 父子组件的访问

父组件访问子组件：$children 或 $refs

  • $children 可以拿到所有的子组件，一般很少用
  • $refs 配置 ref 属性拿到指定的组件，一般使用这种方式
<div id="app">
    <my-component></my-component>
    <button @click="btnClick">按钮</button>
</div>

<template id="myComponent">
    <div>
        <h2>我是子组件</h2>

<script src="../js/vue.js"></script>
<script>
    const vm = new Vue({
        el: '#app',
        data: {},
        methods: {
            btnClick() {
                console.log(this.$children[0])

        components: {
            MyComponent: {
                template: '#myComponent',
                methods: {
                    showMessage() {
                        console.log('hello vue.js');

                    return {
                        name: 'vae'
                    }
                }
            }
        }
    })
</script>

# （1）插槽简介

插槽就是子组件提供给父组件使用的一个占位符，用 表示，父组件可以通过填充任意内容来替换掉 标签。

插槽的作用是为了让我们封装的组件更加具有扩展性，让使用组件的人在使用的时候决定组件内部展示什么内容

插槽的基本使用如下：

<div id="app">
    <my-component></my-component>
    <my-component><p>内容</p></my-component>
    <my-component><span>简单提示</span></my-component>
</div>
<template id="myComponent">
    <div>
        <h2>我是子组件</h2>
        <p>哈哈哈</p>
        <slot><button>插槽的默认内容</button></slot>
    </div>
</template>
<script src="../js/vue.js"></script>
<script>
    const vm = new Vue({
        el: '#app',
        data: {},
        components: {
            MyComponent: {
                template: '#myComponent'
            }
        }
    })
</script>

# （2）具名插槽

具名插槽就是带有名字（name 属性）的插槽。因为有时候我们在组件中需要多个插槽（slot）。这样的情况就需要使用带有名字的插槽（具名插槽）。

场景： 比如导航栏可以封装成一个组件，这样可以多个地方共用，但是不同地方的导航栏左中右每个地方显示的内容不同

<div id="app">
    <my-component>
        <!-- vue2.6 之前的写法, 在 vue2,6 之后被废弃, 
		vue2.x 的后续所有版本都可以使用，在 vue3.x中不可使用-->
        <button slot="left">替换左边的内容</button>
        <!-- vue2.6.0 引入了 v-slot -->
        <template v-slot:right>
            <button>替换右边的内容</button>
        </template>
    </my-component>
</div>
<template id="myComponent">
    <div>
        <slot name="left"><span>左边</span></slot>
        <slot name="center"><span>中间</span></slot>
        <slot name="right"><span>右边</span></slot>
    </div>
</template>
<script src="../js/vue.js"></script>
<script>
    const vm = new Vue({
        el: '#app',
        data: {},
        components: {
            MyComponent: {
                template: '#myComponent'
            }
        }
    })
</script>

# （3）作用域插槽

作用域插槽比较难以理解。。。

（1）描述：作用域插槽其实就是带数据的插槽，即带参数的插槽，简单的来说就是子组件提供给父组件的参数，该参数仅限于插槽中使用，父组件可根据子组件传过来的插槽数据来进行不同的方式展现和填充插槽内容。

（2）场景：如果子组件中的某一部分的数据，每个父组件都会有自己的一套对该数据的不同的呈现方式，这时就需要用到作用域插槽。

（3）使用方法：

  • 子组件存放一个带数据的插槽
  • 父组件通过 slot-scope 来接收子组件传过来的插槽数据，再根据插槽数据来填充插槽的内容

编译作用域：父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的。

<div id="app">
  <!-- 这里的isShow会在父组件(Vue实例)中去找, 父组件中 isShow=true -->
  <my-component v-show="isShow"></my-component>
</div>

<template id="myComponent">
  <div>
    <h2>我是子组件</h2>
    <!-- 这里的isShow会在子组件中去找, 子组件中 isShow=false -->
    <p v-show="isShow">我是内容</p>
  </div>
</template>
<script src="../js/vue.js"></script>
<script>
  const vm = new Vue({
    el: '#app',
    data: {
      isShow: true
    },
    components: {
      MyComponent: {
        template: '#myComponent',
        data() {
          return {
            isShow: false
          }
        }
      }
    }
  })
</script>

作用域插槽的使用：

<div id="app">
  <my-component></my-component>
  <my-component>
    <!-- 使用 template 目的是获取子组件数据 books -->
    <template slot-scope="c1">
      {{c1.data.join('-')}}
    </template>
  </my-component>
  <my-component></my-component>
</div>

<template id="myComponent">
  <div>
    <!-- 子组件中有带参数的slot -->
    <slot :data="books">
      <ul>
        <li v-for="item in books">{{item}}</li>
      </ul>
    </slot>
  </div>
</template>
<script src="../js/vue.js"></script>
<script>
  const vm = new Vue({
    el: '#app',
    data: {},
    components: {
      MyComponent: {
        template: '#myComponent',
        data() {
          return {
            books: ['西游记', '红楼梦', '水浒传', '三国演义']
          }
        }
      }
    }
  })
</script>
