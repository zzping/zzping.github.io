# Vuex（状态管理）

官方文档： https://vuex.vuejs.org/zh/

# Vuex 简介

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。Vuex 也集成到 Vue 的官方调试工具 devtools extension (opens new window) (opens new window)，提供了诸如零配置的 time-travel 调试、状态快照导入导出等高级调试功能。

简单总结： Vuex 是 Vue.js 的一个状态管理的库。状态管理其实就是多个组件共享的变量全部存储在一个对象中；

什么场景下来通过 Vuex 来进行状态管理？

多个组件间需要共享某个数据时候。适合用 Vuex 来进行管理。父子组件这种当然还是用组件间通信简单一点。

# Vuex 核心概念 - State

Vuex 中的 State 就是用来存储多个组件中需要共享的变量的。在 Vuex 中它用 单一状态树这个概念来描述 State。

表示用一个对象就包含了全部的应用层级状态。每个应用将仅仅包含一个 store 实例，这个 store 实例将作为整个项目的单一数据源。所有组件都从这个 store 实例获取数据。

const store = new Vuex.Store({
  state: {
    count: 0 // count 变量用于在多个组件间共享, 放在 state 中
  },
  getters: {},
  mutations: {},
  actions: {},
  modules: {}
})

# Vuex 核心概念 - Getters

有时候我们需要从 store 中的 state 中派生出一些状态， 即对 state 中的共享变量进行一些计算和处理后在使用。这时候就可以使用 Getters， Getters 类似于组件中的计算属性 computed。

const store = new Vuex.Store({
  state: {
    count: 0
  },
  getters: {
      // 一个参数, state
    doubleCount(state) {
      return state.count * 2
    },
      // 两个参数 state、getters
    fourCount(state, getters)  {
      return getters.doubleCount * 2;
    },
      // 想要传入自定义参数 -- 直接返回函数, 使用函数的参数
    dynamicCount(state) {
      return function (n) {
        return state.count * n;
      }
    }
  },
  mutations: {},
  actions: {},
  modules: {}
})

# Vuex 核心概念 - Mutations

更改 Vuex 的 store 中的状态的唯一方法是提交 mutation。mutation 必须是同步函数。

mutation 主要包含两部分：

  • 事件类型 (type)
  • 回调函数 (handler)，该回调函数接收的第一个参数就是 state。

Vuex 中的 mutation 非常类似于事件：每个 mutation 都有一个字符串的 事件类型 (type) 和 一个 回调函数 (handler)。这个回调函数就是我们实际进行状态更改的地方，并且它会接受 state 作为第一个参数。

# （1）Mutations 的基本使用

state: {
count: 1
},
mutations: {
increment(state) {
state.count++
},

# （4）Mutation 的响应式规则

Vuex 的 store 中的 state 是响应式的。当我们通过提交 mutation 更改 state 的时候，对应的视图也会更新。因此这就要求 Mutation 的使用需要遵守一些规则：

(1) 最好提前在你的 store 中初始化好所有所需属性。
(2) 当需要在对象上添加新属性时，你应该使用 Vue.set(obj, 'newProp', 123),
(2) 当需要在对象上删除新属性时，你应该使用 Vue.delete(obj, key),

# Vuex 核心概念 - Actions

Vuex 要求 Mutation 中的方法必须是同步方法，主要原因是当我们使用 devtools 时，devtools 可以帮助我们捕捉Mutation 的快照。如果是异步操作, 那么 devtools 将不能很好的追踪这个操作什么时候会被完成。

Action 类似 Mutation，他们的不同之处在于：

  • Action 中可以包含任意异步操作，Mutation 中只能是同步操作
  • Action 中是通过提交 Muatation 来操作 State， 不能直接操作 State

# Vuex 核心概念 - Modules

由于使用单一状态树，应用的所有状态会集中到一个比较大的对象。当应用变得非常复杂时，store 对象就有可能变得相当臃肿。

为了解决以上问题，Vuex 允许我们将 store 分割成模块（module）。每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块——从上至下进行同样方式的分割：

const moduleA = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... },
  getters: { ... }
}

const moduleB = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... }
}

const store = new Vuex.Store({
  modules: {
    a: moduleA,
    b: moduleB
  }
})

store.state.a // -> moduleA 的状态
store.state.b // -> moduleB 的状态

# Vuex 目录组织结构

（1） Vuex 并不限制你的代码结构。但是，它规定了一些需要遵守的规则：

（1）应用层级的状态应该集中到单个 store 对象中。

（2）提交 mutation 是更改状态的唯一方法，并且这个过程是同步的。

（3）异步逻辑都应该封装到 action 里面。

（2） 对于大型应用，推荐目录结构如下：

├── index.html
├── main.js
├── api
│   └── ... # 抽取出API请求
├── components
│   ├── App.vue
│   └── ...
└── store
    ├── index.js          # 我们组装模块并导出 store 的地方
    ├── actions.js        # 根级别的 action
    ├── mutations.js      # 根级别的 mutation
    └── modules
        ├── cart.js       # 购物车模块
        └── products.js   # 产品模块
