# AOP 细节

# 五大通知注解

@Before： 前置通知，在方法运行之前执行

@After： 后置通知，在方法最终运行结束后执行

@AfterReturning： 返回通知，在方法正常返回后执行

@AfterThrowing： 异常通知，在方法抛出异常后执行

@Around： 环绕通知，围绕整个方法运行

// 其中四个通知就类似于以前动态代理的下面位置。

try {
@Before
method.invoke(obj, args);
@AfterReturning
} catch(Exception e) {
@AfterThrowing
} finally {
@After
}

# Spring 支持两种动态代理

Spring 支持 JDK 动态代理和 CGlib 动态代理。如果目标类实现了接口，则使用 JDK 动态代理，如果目标类没有接口，则使用 CGLIB 代理。

@Service
public class CalculatorImpl /*implements Calculator*/ {

//@Override
public int add(int i, int j) {
  System.out.println("进入 add() 方法了...");
  return i + j;
}

//@Override
public int sub(int i, int j) {
  return i - j;
}

//@Override
public int mul(int i, int j) {
  return i * j;
}

//@Override
public int div(int i, int j) {
  return i / j;
}
}

# 切入点表达式详解

官方文档： 切入点表达式示例 (opens new window)

execution(* *.*(..))  ## 任意返回值、任意包下的任意类中的任意方法, 参数也任意，最简洁的写法

# 抽取可重用的切入点表达式

切面类中为多个通知方法都写切入点表达式比较繁琐，可以使用**@Pointcut**抽取可重用的切入点表达式

@Aspect
@Component
public class LogUtil {

// 切入点： 任意包下任意类的任意方法, 方法的参数任意:  execution(* *.*(..))
@Pointcut("execution(* com.zzp.aop.*Impl.*(..))")
public void myPoint() {
}

// 前置通知
//@Before("execution(public int com.zzp.aop.CalculatorImpl.*(int,int))")
@Before("myPoint()")
public static void logStart() {
  System.out.println("【前置通知】方法开始之前...");
}

// 返回通知
// @AfterReturning("execution(public int com.zzp.aop.CalculatorImpl.*(int,int))")
@AfterReturning("myPoint()")
public static void logReturn() {
  System.out.println("【返回通知】方法正常返回之后...");
}

// 异常通知
// @AfterThrowing("execution(public int com.zzp.aop.CalculatorImpl.*(int,int))")
@AfterThrowing("myPoint()")
public static void logException() {
  System.out.println("【异常通知】方法出现异常后...");
}

// 后置通知
// @After("execution(public int com.zzp.aop.CalculatorImpl.*(int,int))")
@After("myPoint()")
public static void logEnd() {
  System.out.println("【后置通知】方法最终结束之后...");
}

}

# 通知方法的各种参数、绑定异常信息、返回值等

（1）JoinPoint： 连接点，可以获取方法的参数、方法名称、访问修饰符等

@Aspect
@Component
public class LogUtil {

// 切入点： 任意包下任意类的任意方法, 方法的参数任意:  execution(* *.*(..))
@Pointcut("execution(* com.zzp.aop.*Impl.*(..))")
public void myPoint() {
}

// 前置通知
@Before("myPoint()")
public static void logStart(JoinPoint joinPoint) {
  Object[] args = joinPoint.getArgs();
  String name = joinPoint.getSignature().getName();
  System.out.println("【前置通知】" + name + "()方法开始之前,方法的参数：" +     Arrays.asList(args));
}

// 返回通知
@AfterReturning(value = "myPoint()", returning = "result")
public static void logReturn(JoinPoint joinPoint, Object result) {
  String name = joinPoint.getSignature().getName();
  System.out.println("【返回通知】" + name + "()方法正常返回之后,方法的返回值：" + result);
}

// 异常通知
@AfterThrowing(value = "myPoint()", throwing = "ex")
public static void logException(JoinPoint joinPoint, Exception ex) {
  String name = joinPoint.getSignature().getName();
  System.out.println("【异常通知】" + name + "()方法出现异常后,方法的异常信息：" + ex);
}

// 后置通知
@After(value = "myPoint()")
public static void logEnd(JoinPoint joinPoint) {
  String name = joinPoint.getSignature().getName();
  System.out.println("【后置通知】" + name + "()方法最终结束之后");
}

}

# 通知方法的执行顺序

官方文档： Advice Ordering (opens new window)

通知方法的执行顺序分为两种场景：

（1）正常执行： 前置通知、业务方法执行、返回通知、后置通知

（2）发生异常： 前置通知、业务方法执行、异常通知、后置通知

Spring不同版本的切面顺序有差异： Spring 5.2.7 版本之前,返回通知和异常通知会在后置通知之后，5.2.7 版本之后，才改成了正常的顺序（返回通知、异常通知在后置通知之前）

相关文章： Spring AOP 切面通知执行顺序的坑 (opens new window)

# 环绕通知【★】

环绕通知是 Spring 的五大通知之一，功能最强大，一般也是用环绕通知比较多。

@Aspect
@Component
public class LogUtil {

// 切入点： 任意包下任意类的任意方法, 方法的参数任意:  execution(* *.*(..))
@Pointcut("execution(* com.zzp.aop.*Impl.*(..))")
public void myPoint() {
}

// 前置通知
@Before("myPoint()")
public static void logStart(JoinPoint joinPoint) {
  Object[] args = joinPoint.getArgs();
  String name = joinPoint.getSignature().getName();
  System.out.println("【前置通知】" + name + "()方法开始之前, 参数为：" + Arrays.asList(args));
}

// 返回通知
@AfterReturning(value = "myPoint()", returning = "result")
public static void logReturn(JoinPoint joinPoint, Object result) {
  String name = joinPoint.getSignature().getName();
  System.out.println("【返回通知】" + name + "()方法正常返回后, 返回值为：" + result);
}

// 异常通知
@AfterThrowing(value = "myPoint()", throwing = "ex")
public static void logException(JoinPoint joinPoint, Exception ex) {
  String name = joinPoint.getSignature().getName();
  System.out.println("【异常通知】" + name + "()方法出现异常后,异常信息为：" + ex);
}

// 后置通知
@After(value = "myPoint()")
public static void logEnd(JoinPoint joinPoint) {
  String name = joinPoint.getSignature().getName();
  System.out.println("【后置通知】" + name + "()方法最终结束之后");
}


// 环绕通知
@Around(value = "myPoint()")
public Object logAround(ProceedingJoinPoint joinPoint) {
  Object[] args = joinPoint.getArgs();
  String name = joinPoint.getSignature().getName();
  System.out.println("【环绕通知】" + name + "()方法开始之前, 参数为：" + Arrays.asList(args));
  Object result = null;
  try {
      // 执行目标方法
      result = joinPoint.proceed(args);
      System.out.println("【环绕通知】" + name + "()方法正常返回后, 返回值为：" + result);
      // 返回方法执行结果
      return result;
  } catch (Throwable throwable) {
      System.out.println("【环绕通知】" + name + "()方法出现异常后, 异常信息为：" + throwable);
  } finally {
      System.out.println("【环绕通知】" + name + "()方法最终执行结束后");
  }
  return result;
}

}

环绕通知类似于动态代理，joinPoint.proceed（args）方法就类似于动态代理中的 method.invoke（obj, args）方法，可以自定义的在方法执行前、方法执行后、方法出现异常后、方法最终执行结束后这些地方进行自定义逻辑，相当于环绕通知包括了前面4个通知。

环绕通知和其他四个通知的区别：

（1）普通的通知只是在目标方法运行到指定点运行该通知方法，不会影响目标方法的执行，

（2）环绕通知会影响目标方法的执行，比如修改参数、修改返回值等等

# 基于 xml 配置的 AOP

官方文档： 基于 XML 的 AOP 配置 (opens new window)

<!-- 注册bean -->
<bean id="logUtils" class="com.xiao.MyProxy02.LogUtils"/>
<bean id="myCalculator" class="com.xiao.MyProxy02.MyCalculator"/>


<!-- aop 配置 -->
<aop:config>
<!--自定义切面aspect，ref:要引用的类-->
<aop:aspect ref="logUtils">
<!--切入点-->
<aop:pointcut id="point" expression="execution(public int com.xiao.MyProxy02.MyCalculator.*(int,int))"/>
<!--前置-->
<aop:before method="before" pointcut-ref="point"/>
<!--返回-->
<aop:after-returning method="after" pointcut-ref="point" returning="result" />
<!--异常-->
<aop:after-throwing method="exception" pointcut-ref="point" throwing="exception"/>
<!--后置-->
<aop:after method="end" pointcut-ref="point"/>
</aop:aspect>
</aop:config>
