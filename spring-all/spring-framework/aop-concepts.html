# AOP 基本概念和通用术语

AOP： Apspect Oriented Programming，面向切面编程

OOP： Object Oriented Programming，面向对象编程

面向切面编程，基于面向对象编程的基础之上，指的是在程序运行期间，将某段代码动态的切入到指定方法的指定位置进行运行的一种编程方式。

AOP 的核心思想： 在不修改原有代码逻辑的基础上实现功能增强。

# 切面（Aspect）

切面： 由切入点（Pointcut）和通知（Advice）组成。

在 Spring 中切面通常是一个由 xml 配置表示或者是由注解 @Aspect 标注的类。

# 连接点（Joinpoint）

连接点： 指的是程序执行的某个特定位置。

Spring 仅支持方法的连接点， 包括：

  • 方法调用前
  • 方法调用后
  • 方法抛出异常时
  • 方法调用前后等

# 切入点（Pointcut）

切入点： 每个类有多个方法，每个方法又有多个连接点，那些我们感兴趣的、需要切入的点就叫切入点。

在 Spring 中，通过切入点表达式来选择众多连接点中我们需要的切入点。

# 通知（Advice）

通知： 也叫增强，是目标类连接点上的一段程序代码。

Spring 中总共有 5 种通知类型

  • 前置通知（BeforeAdvice）： 在方法执行之前执行，对应注解 @Before
  • 后置通知（AfterAdvice）： 在方法执行之后执行（无论是否发生异常），无方法结果，对应注解 @After
  • 返回通知（AfterReturningAdvice）： 在方法正常结束并返回结果之后执行，对应注解 @AfterRunning
  • 异常通知（ThrowsAdvice）： 在方法抛出异常之后执行，可以访问异常对象，对应注解 @AfterThrowing
  • 环绕通知（AroundAdvice）： 围绕着整个方法执行，相当于动态代理，对应注解 @Around

# 目标对象（Target）

目标对象： 具体的业务逻辑类，需要不修改代码而增强逻辑（增加日志、权限处理等）的类

# 代理对象（Proxy）

代理对象 = 目标对象 + AOP 逻辑（增强逻辑）

代理对象： 既可能是和原类具有相同接口的类，也可能是原类的子类。所以可以采用与调用原类相同的方法调用代理类。
