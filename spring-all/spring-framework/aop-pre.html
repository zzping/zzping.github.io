# AOP 前置知识回顾

# 代理

代理： Proxy，代理模式也是一种常见的设计模式（结构型）。

代理模式： 指的是给目标对象增加一个代理对象，通过代理对象来访问目标对象，这样可以在目标对象的方法执行前后做一些自定义逻辑以实现增强目标对象。

# 静态代理

需求： 在一个已有的业务方法前后增加日志打印。

思路： 为现有的每一个类编写一个对应的代理类。并让代理类实现和目标类一样的接口。让代理类中持有目标类的成员变量，并在创建代理对象的时候通过构造方法传入目标对象赋值给成员变量，然后在代理对象的方法中调用目标对象的方法。并在调用前后增加日志打印。使用的时候也是直接使用代理对象，而不是使用原有目标对象。

（1）编写一个接口 Hello

public interface Hello {

void sayHello(String name);

（5）静态代理总结

优点： 
	- 实现了代理, 在不修改目标对象的前提下, 通过代理对象实现了增强目标对象的功能。
	- 客户端使用的时候只需要通过代理访问目标对象, 不需要知道目标对象的具体实现细节, 实现了解耦。

缺点：
	- 代理类和被代理类(委托类)都实现了相同的接口, 如果接口增加一个方法, 代理类也需增加实现这个方法。
  - 每一个目标类都需要一个对应的代理类。代理类不能通用,会出现很多的代理类,不合理。



静态代理： 在程序运行之前，代理类的字节码 .class 已经生成，通常一个代理类只能代理一个目标类。并且代理类和目标类都必须要实现相同的接口。

静态代理的缺点很明显，每一个代理类只能代理一个目标对象。不能通用。目标类一多就会产生大量的代理类。比较麻烦。

# 动态代理之 JDK 动态代理

为了解决静态代理中要为每一个目标类（委托类）编写一个代理类的问题，就有了 JDK 的动态代理

JDK 的动态代理区别于静态代理的主要有两个点：

  • 静态代理要为每一个目标类编写一个代理类，而JDK动态代理实现了一个通用的代理类代理所有目标类
  • 静态代理中代理类的字节码（.class）在程序运行之前生成。动态代理则是在程序运行时动态生成

JDK 的动态代理主要由反射包中的一个类（Proxy）和一个接口（InvocationHandler）实现：

  • java.lang.reflect.Proxy
  • java.lang.reflect.InvocationHandler

通过 Proxy 类的静态方法 newProxyInstance 方法创建一个动态代理对象

public static Object newProxyInstance(ClassLoader loader,
                                        Class<?>[] interfaces,
                                        InvocationHandler h)

（3）编写一个类实现 InvocationHandler 接口

public class MyInvokeHandler implements InvocationHandler {

  private Object target; // 目标对象

  public MyInvokeHandler(Object target) {
      this.target = target;
  }

  @Override
  public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
      before();
      // 通过反射执行目标对象的目标方法
      Object result = method.invoke(target, args);
      after();
      return result;
  }

  private void before() {
      System.out.println("before...");
  }

  private void after() {
      System.out.println("after...");
  }
}

（4）测试动态代理

public class TestDynamicProxy {

  @Test
  public void test01() {
      Hello hello = new HelloImpl();
      // 实现 InvocationHandler 接口的类就可以看成是通用的代理类。
      // 这个代理类的实例包裹了 Hello 接口的实例。
      InvocationHandler handler = new MyInvokeHandler(hello);


      ClassLoader classLoader = hello.getClass().getClassLoader();
      Class<?>[] interfaces = hello.getClass().getInterfaces();
      // 通过 Proxy.newProxyInstance 创建代理对象
      Hello helloProxy = (Hello) Proxy.newProxyInstance(classLoader, interfaces, handler);

      System.out.println(helloProxy); // com.zzp.dynamicproxy.HelloImpl@58d25a40
      System.out.println(helloProxy.getClass()); // class com.sun.proxy.$Proxy4

      helloProxy.sayHello("小张");
  }
}

# 动态代理之 CGlib 动态代理

JDK 的动态代理解决静态代理中了每一个目标都需要一个代理类的问题，但是没有解决另一个问题： 被代理的目标类必须要实现接口（即静态代理和JDK的动态代理都有一个前提： 目标类需要实现接口）。

很多时候我们的时候目标类是不需要实现接口的，这个时候JDK的动态代理就没法用了。而 CGlib 动态代理可以实现代理没有接口的类。

# 总结

为了能够使一个代理类能够代理所有目标的类以实现通用的代理。这就是动态代理。

JDK 中支持动态代理。它是通过 InvokeHandler 接口和 Proxy 类这两个类配合完成动态代理的

Proxy 和 InvokeHandler 都是在反射包（java.lang.reflect）下

jdk 的动态代理和静态代理比较：


(1) 静态代理： 目标类要实现接口，代理类也要实现一样的接口。这样他们才能产生联系，才能通过代理类调用目标类中的方法。
(2) JDK 动态代理： 目标类要实现接口， 代理类也要实现接口(InvokeHandler)。

静态代理： 代理类的字节码在【程序运行之前】已经生成。
动态代理： 代理类的字节码在【程序运行期间】动态生成。

静态代理： 要为每一个需要代理的类写一个代理类。
动态代理： 只需要写一个代理类，就可以实现一个代理类代理所有目标类(目标类必须要实现接口)
