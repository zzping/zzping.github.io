# 重复注解和类型注解

注解是 JDK5 中新增加的功能, 而在 JDK8 中对注解进行了一些更新。包括新增重复注解和类型注解

# 重复注解

（1）定义一个普通注解

@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
@Repeatable(MyAnnotations.class)
public @interface MyAnnotation {

    String value() default "china";
}

# 类型注解

java8 中为 ElementType 新增了两个枚举值： TYPE_PARAMETER 和 TYPE_USE

  • TYPE_PARAMETER： 表示该注解能使用在自定义类型参数（参数的自定义类型可以是javaBean或者枚举等）的声明语句中
  • TYPE_USE： 表示该注解能使用在使用类型的任意语句中。
// 定义一个简单的 Type Annotation，不带任何成员变量
@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE_USE)
public @interface NotNull {
}

// 使用 Type Annotation
public class TestTypeAnnotation {

    // 声明变量时候使用 Type Annotation
    private @NotNull String name;

    public static void main(String[] args) throws @NotNull Exception{
        Object obj = "fkjava.org";
        // 强制类型转换时使用 Type Annotation
        String str = (@NotNull String)obj;
        // 创建对象时使用 Type Annotation
        Object win = new @NotNull Date();
    }
}

# 总结

java8 中新增的类型注解可以使得注解存在于任何地方。 目的是可以让编译器执行执行更严格的检查，从而提高程序的健壮性。但是需要注意的是这些注解本身并不起任何作用， 应该 还没有提供处理这些注解的工具和框架，目前 Java 本身还没有提供对类型注解的检查框架。可以配合一些第三方插件工具来完成在 Java 程序中做强类型检查，比如使用 Checker Framework， 可以在编译的时候检测到运行时异常【比如空指针异常】，以此提高代码质量。这就是类型注解的作用。

类型注解被用来支持在 java 程序中做强类型检查。
