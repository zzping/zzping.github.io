# 类的加载过程

类加载子系统 Class Loader SubSystem， 它的作用是： 类加载子系统作为 JVM 三层结构中的最上层，它负责从文件系统或者网络中加载 Class 文件，加载的类信息放在方法区的内存空间中（除了类本身的信息之外，方法区中还会存放运行时常量池信息，可能还包括字符串字面量和数字常量）

内存中的运行时常量池对应的是物理磁盘上的 Class 文件中的常量池的部分内存映射。对 Class 文件进行反编译可以看到有常量池 Constant Pool

ClassLoader 只负责 class 文件的加载，至于它是否可以运行，则由 Execution Engine 决定。

# 加载（Loading）

​ 加载（Loading）阶段是整个"类加载（Class Loading）"过程中的第一个阶段，在加载阶段，Java 虚拟机需要完成以下三件事情：

  • 通过一个类的全限定名来获取定义此类的二进制字节流。
  • 将这个字节流所代表的静态存储结构转化为方法区的运行时数据结构。
  • 在内存中生成一个代表这个类的java.lang.Class 对象，作为方法区这个类的各种数据的访问入口。

这里的内存实际上是一个逻辑结构，在不同版本的 JDK 中有不同的物理实现。

在 JDK7 及以前，HotSpot 使用永久代来实现方法区

在 JDK8 及以后，HotSpot 使用元空间来实现方法区

​ 加载阶段结束后 ，Java 虚拟机外部的二进制字节流就按照虚拟机所设定的格式存储在方法区之中了，方法区中的数据存储格式完全由虚拟机实现自行定义，《Java 虚拟机规范》未规定此区域的具体数据结构。类型数据妥善安置在方法区之后，会在 Java 堆内存中实例化一个 java.lang.Class 类的对象，这个对象将作为程序访问文件中的类型数据的外部接口。

加载阶段和链接阶段的部分动作（如一部分字节码文件格式验证动作）是交叉进行的，加载阶段尚未完成，链接阶段可能已经开始，但这些夹在加载阶段之中进行的动作，仍然是属于链接阶段的一部分，这两个极端的开始时间仍然保持着固定的额先后顺序。

# 链接（Linking）

链接（Linking）是类加载过程中的第二个阶段，它具体由分为了三个阶段：验证、准备、解析

# （1）验证 - Verification

验证是链接阶段的第一步，验证的目的是为了确保 Class 文件的字节流中包含的信息符合《Java虚拟机规范》的全部约束要求，保证这些信息被当做代码运行后不会危害虚拟机自身的安全。

验证阶段大致分为四个部分的校验动作：文件格式验证、元数据验证、字节码验证、符号引用验证

（1）文件格式验证

  • 是否以魔数 0xCAFEBABE 开头
  • 主次版本号是否在当前 Java 虚拟机接受范围之内
  • ......

（2）元数据验证

  • 这个类是否有父类（除了 java.lang.Object 外，所有的类都应当有父类）
  • 这个类的父类是否继承了不允许被继承的类 （被 final 修饰的类）
  • ......

（3）字节码验证

（4）符号引用验证

# （2）准备 - Preparation

准备阶段是正式为类中定义的变量（即静态变量，被 static 修饰的变量）分配内存并设置类变量初始值的阶段。从概念上讲。这些类变量所使用的内存都应当在方法区中进行分配，但必须注意到方法区本身是一个逻辑上的概念，在 JDK7 及以前，HotSpot 使用永久代来实现方法区时，实现是完全符合这种逻辑概念的；而在 JDK8 及以后，类变量则会随着 Class 对象一起存放在 Java 堆中，这时候 "类变量在方法区"就完全是一种堆逻辑概念的表述了。

public static int num = 100;
1

在上面定义的类变量中，经过准备阶段后，num 的值是 0。

Java 中各种数据类型的初始值（零值）如下：

数据类型 初始值（零值）
byte (byte)0
short (short)0
int 0
long 0L
char '\u0000'
boolean false
float 0.0f
double 0.0d
reference null

特殊情况：被 final 修饰的类变量在经过 javac 编译后会为 num 生成 ConstantValue 属性，在准备阶段虚拟机就会根据 ConstantValue 的设置将 num 赋值为 200。

public static final int num = 200
1

在上面定义的类变量中，经过准备阶段后，num 的值是 200。

# （3）解析 - Resolution

解析阶段是 Java 虚拟机将常量池内的符号引用替换为直接引用的过程。

符号引用： 以一组符号来描述所引用的目标，符号可以是任何形式的字面量。

直接引用： 直接引用可以是直接指向目标的指针、相对偏移量或者是一个能间接定位到目标的句柄。

# 初始化（Initialization）

初始化是类加载的最后一个阶段，也是 Java 虚拟机正式执行类中编写的 Java 程序代码的阶段。

初始化阶段就是执行类构造器 &lt;clinit&gt;() 方法的过程。<clinit>( ) 方法并不是 Java 代码中直接编写的方法，它是 javac 编译器的自动收集类中所有的类变量的赋值动作和静态语句块中的语句合并产生的。编译器收集的顺序是由语句在源文件中出现的顺序决定的。

&lt;clinit&gt;() 方法与类的构造方法（即在虚拟机视角中的实例构造器 &lt;init&gt;() 方法）不同。

<clinit>() 方法对于类或者接口来说 并不是必须的，如果一个类中没有静态代码块，也没有对静态变量的赋值操作，那么编译器就不会为这个类生成 <clinit>() 方法。

1、 javac 编译器收集静态语句块和给静态变量赋值的语句是按照语句出现在源文件中的顺序进行收集的

public class Test {
   private static int a = 10;
   static {
       a = 20;
   }

   public void static main(String[] args) {
       System.out.println(a) // 20
   }
}


public class Test {
   static {
       a = 10;
   }
   private static int a = 100;

   public void static main(String[] args) {
       System.out.println(a) // 100
   }
}

   static {
       num = 20;  // 可以给定义在静态代码块之前的变量赋值

   }

   public static void main(String[] args) {
       System.out.println(num);
   }
}
