# Mybatis 整体架构

# 整体三层架构简介

Mybatis 整体架构分为三层：

  • 基础支撑层

  • 核心处理层

  • 接口层

Mybatis 的源码中基本上也是一个 package 对应一个模块：

# 基础支撑层介绍

基础支撑层包含了整个 Mybatis 的基础模块，这些基础模块为核心处理层的功能提供了良好的支撑。

模块名称 对应的 package 作用
反射模块 reflection 对原生反射进行封装，提供更强大、方便的调用
类型转换模块 type 1、实现JDBC类型和Java类型相互转换；2、提供别名机制
日志模块 logging 集成第三方日志框架，提供日志处理功能
资源加载模块 io 对类加载器进行封装，提供加载各种资源文件的功能
解析器模块 parsing 解析xml配置文件
数据源模块 datasource 提供了数据源实现，也提供了和第三方数据源集成的接口
事物管理模块 transaction 自身提供了事物接口和简单实现，支持和Spring集成
缓存模块 cache 持久层框架都少不了缓存处理
Binding 模块 binding 映射绑定，将用户自定义的Mapper接口与映射配置文件关联

# 核心处理层介绍

核心处理层实现了 Mybatis 的核心处理流程。包括： Mybatis 的初始化和一次操作数据库涉及的全部流程。

模块名称 对应的 package 作用
模块解析 builder 解析Mybatis的全局配置文件和SQL映射文件，包括xml和注解两种形式
参数映射 mapping 主要处理 ParameterMap，对输入参数的判断、组装等
SQL解析 scripting 根据传入参数解析SQL映射文件中的动态SQL、占位符等处理成真正SQL
SQL执行 executor 在SQL解析完成后执行SQL语句并将结果返回
结果集映射 mapping 主要处理 ResultMap 等，和 ParameterMap 类似
插件 plugin 可以通过自定义插件对 Mybatis 进行功能扩展，例如： Mybatis Helper

# 接口层介绍

接口层核心是 SqlSession 接口，提供了一系列操作数据的增删改查的 API。当接口层收到调用请求时会调用核心处理层的模块来配合完成具体的数据库操作。

模块名称 对应的 package 作用
session session 提供一系列操作数据库的增删改查API
