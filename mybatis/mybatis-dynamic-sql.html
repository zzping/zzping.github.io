# Mybatis 动态 SQL

# 动态 SQL 简介

动态 SQL 是 MyBatis 的强大特性之一。如果你使用过 JDBC 或其它类似的框架，你应该能理解根据不同条件拼接 SQL 语句有多痛苦，例如拼接时要确保不能忘记添加必要的空格，还要注意去掉列表最后一个列名的逗号。利用动态 SQL，可以彻底摆脱这种痛苦。

在 Mybatis 早期的版本中，使用动态 SQL 需要学习和了解非常多的标签，现在 Mybatis 采用了功能强大的 OGNL 表达式语言消除了许多其他标签，只需要掌握极少的标签就能使用动态 SQL。

Mybatis 的动态 SQL 包含以下几种元素：

if：判断，单条件分支判断

choose（when、otherwise）：判断，多条件分支判断，相当于 Java 中的 case when 语句

trim（where、set）： 辅助元素，用于处理一些 SQL 拼接问题

foreach：循环，常用语 in 等列举语句中

# choose、when、otherwise

if 能够实现基本的条件判断，相当于 java 中的 if，但是无法实现 if、else if、else 的逻辑。要想实现这样的逻辑，就需要用到 choose、when、otherwise，choose 标签中包含了 when 和 otherwise 标签。choose 标签相当于 java 中带了 break 的 switch 语句：

  • choose： 相当于 java 中的 switch
  • when： 相当于 java 中的 case
  • otherwise： 相当于 java 中的 default

需求： 查询用户，带了 id 就通过id查询，带了 lastName 就通过 lastName，都没带则查询所有女生

public interface DynamicSqlMapper {

   // 动态 sql - choose(when、otherwise)
   List<User> getUserByConditionChoose(@Param("user") User user);
}

<!-- 动态 sql - choose(when、otherwise) -->
<select id="getUserByConditionChoose" resultType="com.zzp.entity.User">
select * from t_user
where
<choose>
  <when test="user.id != null">
      id = #{user.id}
  </when>
  <when test="user.lastName != null and user.lastName != ''">
      last_name = #{user.lastName}
  </when>
  <otherwise>
      gender = 0
  </otherwise>
</choose>
</select>

# where、set、trim

（1）where 元素

where 标签的作用： 如果该标签包含的元素中有返回值，就插入一个 where，如果 where 后面的字符串是以 and 或者 or 开头的，就将它们剔除。

<select id="getUserByConditionIf" resultType="com.zzp.entity.User">
select * from t_user
<where>
  <if test="id != null">
      id = #{id}
  </if>
  <if test="lastName != null and lastName != ''">
      and last_name like concat("%", #{lastName}, "%")
  </if>
</where>
</select>

（3）trim 元素

where 和 set 标签的功能都可以使用 trim 标签来实现。在底层是通过 TrimSqlNode 实现的。trim 标签有如下的4 个属性：

  • prefix：当 trim 元素内包含内容时，会给内容增加 prefix 指定的前缀
  • prefixOverrides： 当 trim 元素内包含内容时，会把内容中匹配的前缀字符串去掉
  • suffix：当 trim 元素内包含内容时，会给内容增加 suffix 指定的后缀
  • suffixOverrides： 当 trim 元素内包含内容时，会把内容中匹配的后缀字符串去掉

# foreach

foreach用于对集合进行遍历，典型的使用场景就是构建 IN 条件语句的时候。

foreach 功能非常强大，它可以遍历（List、Set、Map、数组）

-	遍历 List、Set、数组时： index 是当前迭代的序号，item 的值是本次迭代获取到的元素。
-	遍历 Map 对象时： index 是键，item 是值

foreach 标签有如下属性：

  • collection： 表示要遍历的集合，可以是数组、List、Set 或者 Map。
  • item： 循环中的当前元素
  • index： 当前元素在集合中的位置下表
  • open：给遍历出所有结果拼接一个开始的字符
  • close：给遍历出所有结果拼接一个结束的字符
  • separator： 每个元素之间的分隔符

# bind

bind 元素可以将 OGNL 表达式的值绑定到一个变量中，方便后来引用这个变量的值。

<select id="selectBlogsLike" resultType="Blog">
<bind name="pattern" value="'%' + _parameter.getTitle() + '%'" />
SELECT * FROM BLOG
WHERE title LIKE #{pattern}
</select>
