# Mybatis 缓存机制

官方文档： Mybatis 缓存 (opens new window)

美团技术团队： 聊聊 Mybatis 缓存 (opens new window)

# Mybatis 缓存介绍

对于持久层框架，缓存是一定需要考虑的。缓存可以极大的提升查询效率。

Mybatis 有两级缓存： 一级缓存和二级缓存

一级缓存： SqlSession 级别的，也称为本地缓存，默认是开启的。

二级缓存： namespace 级别的，需要手动开启和配置。

除此之外，Mybatis 定义了缓存接口 Cache，我们可以通过实现 Cache 接口来自定义二级缓存。

# （2）一级缓存测试

@Test
public void test01() {
   SqlSessionFactory sqlSessionFactory = getSqlSessionFactory();
   try(SqlSession sqlSession = sqlSessionFactory.openSession()) {
       UserMapper mapper = sqlSession.getMapper(UserMapper.class);

       User user01 = mapper.getUserById(1);
       // 一级缓存就是一个 HashMap, 对应的类是 PerpetualCache(永久缓存,不会过期)
       User user02 = mapper.getUserById(1);
       System.out.println(user01);
       System.out.println(user02);
       System.out.println(user01 == user02); // true
   }
}

# （3）一级缓存失效情况

  • sqlSession 不同
  • sqlSession 相同，查询条件不同（当前一级缓存中还没有这个数据）
  • sqlSession 相同，两次查询之间执行了增删改操作（这次增删改可能对当前数据有影响）
  • sqlSession 相同，手动清楚了一级缓存（sqlSession.clearCache()）

# 二级缓存（namespace 级别）

官方文档： Mybatis 缓存 (opens new window)

# （1）二级缓存简介

Mybatis 的二级缓存是基于 namespace 的，默认是不开启的，如果需要开启二级缓存，需要手动进行配置。

（1）全局配置文件中配置开启全局缓存

<settings>
<!-- 配置开启二级缓存, cacheEnabled 默认值也是 true, 可以省略 -->
<setting name="cacheEnabled" value="true"/>
<!-- 其他的配置 ... -->
</settings>

1

开启二级缓存后的效果：

(1) 映射语句文件中的所有 select 语句的结果将会被缓存。
(2) 映射语句文件中的所有 insert、update 和 delete 语句会刷新缓存。
(3) 缓存会使用最近最少使用算法（LRU, Least Recently Used）算法来清除不需要的缓存。
(4) 缓存不会定时进行刷新（也就是说，没有刷新间隔）。
(5) 缓存会保存列表或对象（无论查询方法返回哪种）的 1024 个引用。
(6) 缓存会被视为读/写缓存，这意味着获取到的对象并不是共享的，可以安全地被调用者修改，而不干扰其他调用者或线程所做的潜在修改。

（4）测试二级缓存

查出的数据都会默认被先放在一级缓存中，只有会话（SqlSession）提交或者关闭以后，一级缓存中的数据才会被放到二级缓存中。

@Test
public void test02() {
   SqlSessionFactory sqlSessionFactory = getSqlSessionFactory();
   // 第一个 SqlSession
   SqlSession sqlSession1 = sqlSessionFactory.openSession();
   UserMapper mapper1 = sqlSession1.getMapper(UserMapper.class);
   User user01 = mapper1.getUserById(1);
   System.out.println(user01);
   sqlSession1.close();

   // 第二个 SqlSession
   SqlSession sqlSession2 = sqlSessionFactory.openSession();
   UserMapper mapper2 = sqlSession2.getMapper(UserMapper.class);
   User user02 = mapper2.getUserById(1);
   System.out.println(user02);
   sqlSession2.close();

   // 如果 cache 的 readOnly 属性为 true, 这里返回 true, 反之返回 false
   System.out.println(user01 == user02); 
}

# （3）二级缓存相关的设置和属性

（1）cache 标签有很多属性可以进行配置，具体如下：

<cache
eviction="FIFO"
flushInterval="60000"
size="512"
readOnly="true"/>
1
2
3
4
5
(1) eviction: 缓存的回收策略(清除策略), 默认的是 LRU, 可用的回收策略如下：
		- LRU: 最近最少使用, 移除最长时间不被使用的对象。
		- FIFI: 先进先出, 按照对象进入缓存的顺序来移除它们。
		- SOFT: 软引用, 基于垃圾回收器状态和软引用规则移除对象。
		- WEAK: 弱引用, 更积极地基于垃圾收集器状态和弱引用规则移除对象。
(2) flushInterval: 缓存刷新间隔, 任意正整数, 单位毫秒, 默认情况是不设置, 表示默认不清空缓存。
(3) size: 缓存的引用个数, 要注意欲缓存对象的大小和运行环境中可用的内存资源。默认值是 1024
(4) readOnly: 是否只读, 默认 false
		- true: 只读, 只读缓存的会返回相同实例, 速度快, 但是这些对象不能改变, 所以可能不安全。
				mybatis 认为所有从缓存中获取数据的操作都是只读的，不会修改数据。所以 mybatis 
				为了加快速度，直接回返回数据的引用。
		- false:  非只读, 可读写的缓存会通过序列化返回缓存对象的拷贝, 速度稍慢, 但是安全。
		        mybatis 认为所有从缓存中获取的数据可能会被修改。
(5) type: 指定实现了 Cache 接口的自定义缓存的全类名, 不使用默认提供的缓存实现。

（2）cacheEnabled 属性

cacheEnabled 控制的是二级缓存的总开关, cacheEnabled 的值不会影响一级缓存。一级缓存默认都是开启的。
1

（3）select 标签的 useCache 属性

useCache: 默认值为 true, 表示本条 select 语句的查询结果会被二级缓存缓存起来。
useCache 属性控制的也是二级缓存, 不影响一级缓存, 只不过它控制的是当前这个 select 标签的查询的二级缓存。
1
2

（4）insert、update、delete 标签的 flushCache 属性

增删改会清空一级缓存和二级缓存。原因是因为增删改标签的 flushCache 的值默认为 true
1

（5）SqlSession.clearCache 清除缓存

SqlSession.clearCache 只是清除当前 sqlSession 的一级缓存
1

（6）全局设置 localCacheScope（设置一级缓存作用域）

localCacheScope: 

可选值： SESSION 、STATEMENT
默认值： SESSION

官方解释：  MyBatis 利用本地缓存机制（Local Cache）防止循环引用和加速重复的嵌套查询。 默认值为 SESSION，会缓存一个会话中执行的所有查询。 若设置值为 STATEMENT，本地缓存将仅用于执行语句，对相同 SqlSession 的不同查询将不会进行缓存。

# 实际开发中的缓存使用

实际项目中不会使用 Mybatis 的二级缓存，缓存一般还是基于业务设计比较好。

基于业务做缓存，而不是基于 SQL 做缓存。

Mybatis 的一级缓存和二级缓存在实际开发中都没有太大意义。

生产环境使用 Mybatis 建议：

（1）关闭 Mybatis 缓存（一级缓存 + 二级缓存），只把 Mybatis 当成一个 ORM 框架来用。
